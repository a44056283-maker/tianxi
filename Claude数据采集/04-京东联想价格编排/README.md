# 京东联想价格编排

中文任务名：京东联想价格编排  
taskName：`daily-jd-lenovo-price-sync`

## 任务定位

这是任务一的正式日采任务。

它必须完成三层工作：

1. 对 `库存 > 0` 的 SKU 做当日真实价格复核
2. 对错链 / 失效链 / 旧价 / 空白显示做当轮收口
3. 对前端零售区做当轮显示校验，不允许“采到了但前端还是空白”

它不是只读编排说明，也不是只跑脚本重建。

## 原始输入

放到：

- `原始输入/`

可包含：

- 手工补录批次
- 页面截图
- 详情页价格证据

## 固定执行顺序

1. 先读取当日 `库存 > 0` SKU 清单
2. 按 SKU 全量遍历，不允许只挑几条样例
3. 对每个 SKU 固定按以下顺序执行：
   - 京东指定店
   - 指定店确实没有，再京东全站
   - 联想官旗
   - 官旗确实没有或已下架，再天猫 / 淘宝替代
4. 对每个命中的商品，必须逐项核对：
   - 系列
   - CPU
   - 内存
   - 硬盘
   - 显卡
   - 尺寸 / 屏幕
   - 颜色
5. 只要任一关键参数不一致：
   - 该链接作废
   - 不写价格
   - 继续按固定顺序补采下一来源
6. 采集完成后统一执行：
   - `build:product-url-locks`
   - `build:collection-plan`
   - `audit:retail-prices`
   - `build:retail-zone`
7. 最后必须检查前端：
   - 京东价是否显示
   - 官旗价是否显示
   - 天猫替代价是否显示
   - 错链是否已经替换
   - 不允许采到数据后前端仍显示空白

## 成功标准

1. 当轮 `库存 > 0` SKU 全部进入任务范围
2. 已采到的 SKU 必须在前端显示对应来源价或替代来源价
3. 错链、失效链、旧价当轮就要替换，不允许继续挂在前端
4. 若某个 SKU 京东、官旗、天猫 / 淘宝都没有有效详情页，才允许写阻塞
5. 没有完成上述收口，不得把任务一写成 `已真实完成`

## 硬门槛

以下任一条件未满足，任务一一律只能写成 `已执行但未收口` 或 `阻塞`：

1. `库存 > 0` SKU 没有全部进入当轮任务范围
2. `missingPriceCount > 0`
3. `missingLinkCount > 0`
4. 前端仍有已在库 SKU 不显示京东价 / 官旗价 / 天猫替代价
5. 错链、失效链、旧价仍留在前端
6. 只做了重建，没有做真实页面采集
7. 只做了部分 SKU，没有把全量队列收完
8. 前端任何在库 SKU 仍出现 `待采集`、`待审批`、`待匹配` 等未完成口径

只有当轮全量在库 SKU 都完成“采集 -> 写回 -> 重建 -> 前端显示”后，才允许写 `已真实完成`。

## 绝对执行规则

1. 不管价格最终来自京东、联想官旗、天猫还是淘宝，只要已经采到有效价格，就不允许前端继续出现 `待采集`、`待审批`、`待匹配`。
2. 任务一一旦开始，未采完全部在库 SKU 前不得主动停止。
3. 必须按当天实时页面执行，不允许拿历史价格、旧链接、旧快照冒充当天完成。
4. 任何“脚本跑过”“解释过”“部分机型已补”都不能替代全量实时采集完成。

## 加强版禁止项

以下行为在任务一中一律视为执行错误：

1. 只采几条样例，就停止当轮任务
2. 看到前端空白，却不回到采集层和决策层收口
3. 已有采集记录，但不把价格和链接正常显示到前端
4. 先谈审批、先谈解释、先谈替代逻辑，而不是先把京东主链跑完
5. 发现京东指定店没有，就不继续转京东全站
6. 发现官旗没有，就不继续转天猫 / 淘宝
7. 把搜索页 / 入口页 / 错商品页当成真实详情页
8. 用“已执行但未收口”掩盖本该继续做完的采集工作
9. 把定时任务做成“脚本触发过”，而不是“全量在库 SKU 当日真实收口”
