🔑 步骤二：API Key 鉴权（生产环境必开）

当前模式: --。开发环境可以不开；生产/跨网络必须开。

设置 API Key（本机操作）：

# 在 api-server 启动前设置环境变量 export COLLECTION_BRIDGE_API_KEY="<你的随机 token>" # 例：openssl rand -hex 32 # 启动 api-server launchctl load ~/Library/LaunchAgents/com.lenovo-smart-retail.api-server.plist

在 plist 中固化：编辑 ~/Library/LaunchAgents/com.lenovo-smart-retail.api-server.plist，加 EnvironmentVariables：

<key>EnvironmentVariables</key> <dict> <key>COLLECTION_BRIDGE_API_KEY</key> <string>替换为你的 token</string> </dict>

另一台电脑调用时在 HTTP Header 中加：

X-Bridge-Token: 替换为你的 token

📤 步骤四：提交凭证（核心）

另一台电脑拍照后调用 /api/collection/v1/submit，传入凭证图片 + 元数据。

请求格式（multipart/form-data）

POST -- Content-Type: multipart/form-data; boundary=... X-Bridge-Token: 你的 token --boundary Content-Disposition: form-data; name="file"; filename="voucher.jpg" Content-Type: image/jpeg <图片二进制> --boundary Content-Disposition: form-data; name="metadata" { "staff_id": "EMP005", "scan_type": "single_scan", "source_group": "智店通入库群", "client_tag": "laptop-b-collect-v1", "customer_name": "张三", "customer_phone": "13800138000", "serial_number": "BH022VR7", "order_number": "XS26060821445335779" } --boundary--

metadata 字段说明

字段必填类型说明
staff_id必填string员工 ID（EMP003/EMP004/EMP005/EMP006）
scan_type必填string扫法：single_scan / multi_scan / three_piece / two_piece
source_group可选string来源群，默认 "智店通入库群"
customer_name可选string客户姓名（OCR 也会提取）
customer_phone可选string客户手机号
agent_phone可选string代扫电话
serial_number可选stringSN，匹配 sales_order_line 优先
order_number可选string订单号
education_discount_amount可选number教育补贴金额（默认 30 单扫 / 500 三件套）
service_fee_per_unit可选number代扫服务费（默认 50 单扫 / 300 三件套）
zhixiangjin_amount可选number智享金（默认 0 / 2000 三件套）
client_tag可选string采集端来源标识，方便审计
notes可选string备注

响应格式

{ "ok": true, "record_id": "BRIDGE-1780999240404-91941F16", "status": "success", // success | partial | failed "matched": true, "match_source": "serial", // serial | order | phone | null "matched_sku": "20003365", "matched_product": "Lenovo Care 智惠 4年", "warnings": [], "errors": [], "evidence_path": "今日水印相机/processed/2026-06-09/EMP005/xxx.jpg", "processing_time_ms": 325, "server_time": "2026-06-09T18:00:40+08:00" }

Python 3（另一台电脑跑）

import requests API_BASE = "--" API_KEY = "你的 token" # 本机 COLLECTION_BRIDGE_API_KEY headers = {"X-Bridge-Token": API_KEY} # 1. 健康检查 resp = requests.get(f"{API_BASE}/api/collection/v1/health") print(resp.json()) # 2. 获取员工列表 resp = requests.get(f"{API_BASE}/api/collection/v1/staff", headers=headers) print(resp.json()) # 3. 提交凭证（核心） with open("voucher.jpg", "rb") as f: files = {"file": ("voucher.jpg", f, "image/jpeg")} data = {"metadata": json.dumps({ "staff_id": "EMP005", "scan_type": "single_scan", "source_group": "智店通入库群", "client_tag": "my-collect-app-v1", "customer_name": "张三", "customer_phone": "13800138000", "serial_number": "BH022VR7", })} resp = requests.post( f"{API_BASE}/api/collection/v1/submit", headers=headers, files=files, data=data, ) result = resp.json() print(f"record_id={result['record_id']} status={result['status']}") # 4. 用 SN 预匹配（采集前先用 SN 试一下能不能匹配到 SKU） resp = requests.post( f"{API_BASE}/api/collection/v1/match/serial/BH022VR7", headers=headers, ) print(resp.json())

Node.js（另一台电脑跑）

const fs = require('fs'); const FormData = require('form-data'); const API_BASE = '--'; const API_KEY = '你的 token'; // 1. 健康检查 const healthResp = await fetch(`${API_BASE}/api/collection/v1/health`); const health = await healthResp.json(); console.log(health); // 2. 提交凭证 const form = new FormData(); form.append('file', fs.createReadStream('voucher.jpg')); form.append('metadata', JSON.stringify({ staff_id: 'EMP005', scan_type: 'single_scan', source_group: ''\u667a\u5e97\u901a\u5165\u5e93\u7fa4', client_tag: 'my-collect-app-v1', customer_name: '\u5f20\u4e09', customer_phone: '13800138000', serial_number: 'BH022VR7', })); const resp = await fetch(`${API_BASE}/api/collection/v1/submit`, { method: 'POST', headers: { 'X-Bridge-Token': API_KEY, ...form.getHeaders() }, body: form, }); const result = await resp.json(); console.log(`record_id=${result.record_id} status=${result.status}`);

🔍 常见问题排查

连接不上本机

  • 检查两台电脑是否在同一 WiFi（不能用手机热点给另一台电脑）
  • 本机防火墙 → 系统设置 → 网络 → 防火墙 → 关闭
  • 本机执行 lsof -i :8000 确认 api-server 在监听
  • 本机先自测 curl http://127.0.0.1:8000/api/collection/v1/health
  • 路由器是否开启了 AP 隔离（会导致同一 WiFi 下电脑互不可见）

403 Invalid X-Bridge-Token

  • 检查另一台电脑 Header 中的 token 与本机环境变量 COLLECTION_BRIDGE_API_KEY 一致
  • 检查没有多余空格 / 换行

OCR 识别不到字段

  • 本机 OCR service (8765) 必须启动：curl http://127.0.0.1:8765/health
  • 检查图片清晰度（手机拍照建议白天自然光）
  • 凭证模板要清晰：客户名 / 客户电话 / 订单号 / SN / 金额

未匹配到销售流水

  • SN 不在数据库中：可能是没入系统的新机
  • 订单号格式不对：要 XS 开头的智店通订单号
  • 客户手机号匹配暂未支持（数据库结构限制）
  • 匹配失败 record 也会入库，状态为 partial，事后人工补

🆘 应急 / 降级

如果对接不通，另一台电脑可以走"降级流程"：

  • 凭证图片放到本机的 /Volumes/TianLu_Storage/Shared/今日水印相机/incoming/
  • 本机的 watermark_camera_sync.py 守护进程会自动 OCR + 匹配 + 入库
  • 这个降级链路 5 分钟扫描一次
