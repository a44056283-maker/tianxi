采集流程

代扫员按以下流程完成一条教育补贴记录：

1. 客户出示教育券 → 2. 智店通出库绑定 SN → 3. 群内上报 → 4. 同步至代扫群页 → 5. 财务核对 + 收款

每一步骤都有对应的字段录入要求，详见下方字段说明。

双群分类规则

系统按 sourceGroupName 字段将记录分为两类：

群 标识 服务费 典型场景
智店通入库群 智店通 ¥50 / 单 出库单 + SN + 教育券同轮证据完整
教育补贴群 教育补贴 ¥30 / 单 教育券图片查看器实扫，按普通单扫正式入库

字段口径

三件套 / 两件套判定

营销库中存在以 MTM 维度匹配的"三件套"和"两件套"组合时，系统会自动标记 bundleChargeApplied: true 并额外计算 zhixiangjinAmount。

  • 三件套：笔记本 + 平板 + 手机（或 同价位替换）
  • 两件套：笔记本 + 平板 / 手机
  • 普通单扫：单件笔记本，不命中套装的场景

套装命中后，serviceFeePerUnit 仍按单台计算，但 zhixiangjinAmount 按营销库规则累加。代扫员录入时需在手机端表单的"备注"里注明套装细节。

常见问题

客户电话与代扫电话不一致怎么办？

属于正常情况。系统会在 latest-education-agent-scan-phone-mismatch-alerts.json 里记录告警。代扫员录入时需填写真实代扫电话，不要直接复制客户电话。

服务费何时到账？

服务费按月结算，财务核对后由店长统一转账到代扫员微信 / 支付宝。状态从 未付 变为 已收款 由管理端手动标记。

教育券码如何核销？

代扫员在智店通会员页扫码核销券码，voucherCode 和 voucherVerifiedAt 由智店通回调自动填充。如果券码核销时间晚于出库时间 1 小时以上，需在备注里说明原因。

本地记录如何与远端数据合并？

管理端启动时会从 latest-education-subsidy-agent-scan-summary.json 加载远端记录，并叠加 localStorage 中的本地记录。点击"同步至代扫群"会把本地记录写入 localStorage 的 edu_subsidy_last_sync_payload 键，供后续 Codex 抓取。

如何清理本地记录？

在手机端长按记录或点 🗑 按钮可删除单条；管理端不提供删除功能（避免误删远端）。若需清空所有本地记录，浏览器控制台执行 localStorage.removeItem('edu_subsidy_local_records_v1')。

Codex 抓取指南

Codex 在审计教育补贴数据时，可使用以下结构化字段：

  • 本地记录持久化键：edu_subsidy_local_records_v1
  • 最近同步载荷键：edu_subsidy_last_sync_payload
  • 载荷结构：{ syncedAt, source, localCount, groupBreakdown, records, totalServiceFee, totalDiscount }
  • 远端源文件：apps/web-cockpit/public/data/latest-education-subsidy-agent-scan-summary.json
  • Sync Gap 告警：latest-education-agent-scan-sync-gap.json
  • 电话不匹配告警：latest-education-agent-scan-phone-mismatch-alerts.json

所有字段均为 UTF-8 JSON，可直接解析。Codex 审计时优先检查 status='未付' 的记录和服务费差额。

本地 OCR 服务（v2 自动识别）

员工手机端上传凭证后自动调用本地 OCR 服务（基于 rapidocr_onnxruntime），提取订单号、客户名、手机号、金额、SN、券码等字段。

  • 服务脚本：scripts/ocr_service.py（独立 FastAPI）
  • 默认端口：127.0.0.1:8765
  • 启动命令：python3 scripts/ocr_service.py --host 127.0.0.1 --port 8765
  • 接口：
    • GET /health - 健康检查
    • POST /ocr/upload - 上传图片，返回原始 OCR 行
    • POST /ocr/extract - 上传图片，返回提取的字段
    • POST /ocr/batch - 批量上传多张图片，合并字段
  • 提取字段：customerName / customerPhone / agentPhone / orderNumber / serialNumber / voucherCode / educationDiscount / serviceFee / zhixiangjin / scanType / extractionConfidence
  • 正则匹配：手机号 11位 / 订单号 XS 开头 / 券码 16位 / 金额 ¥/元/圆
  • 扫法判定：基于文本关键词 "三件套/青春有AI" / "两件套/锦鲤跃龙门" / "双屏/拯救者"
  • 降级机制：OCR 不可用时降级为本地 mock 模式（不会阻塞流程）
