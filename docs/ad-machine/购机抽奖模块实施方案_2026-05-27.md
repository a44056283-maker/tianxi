# 购机抽奖模块实施方案

更新时间：2026-05-27  
适用范围：`广告机首页独立入口 -> 购机抽奖模块 -> 手机扫码留资页 -> 后台管理端 -> SQL 数据同步`

## 1. 目标

本模块用于在广告机首页独立入口中承接“购机抽奖”互动，满足以下目标：

1. 正式抽奖必须以 `SN` 为唯一凭证。
2. 无 `SN` 时允许模拟抽奖，增强客户互动性。
3. 抽奖结果、参与记录、奖池消耗需要同步至后台管理端与 SQL 数据库。
4. 奖池必须按轮次耗尽后才能切换下一池。
5. 客户端只看到规则、玩法、结果与公示，不暴露后台奖励分配逻辑。

## 2. 总体玩法

模块分为两条链路：

### 2.1 正式抽奖链路

1. 客户购机后取得 `SN`。
2. 客户扫描广告机抽奖页二维码。
3. 手机页填写：姓名、手机号、购机机型、`SN`。
4. 系统校验 `SN` 资格。
5. 广告机显示“待抽奖客户”状态。
6. 店员或客户点击转盘中心按钮执行正式抽奖。
7. 抽奖结果上屏，并生成后台可核对的记录。
8. 奖池扣减一张奖卡。
9. 结果同步 SQL 与后台管理平台。

### 2.2 模拟抽奖链路

1. 客户无需 `SN`，点击“模拟抽奖”。
2. 可选填写昵称或机型，也可匿名体验。
3. 执行模拟转盘动画。
4. 生成体验结果，仅用于现场互动。
5. 模拟结果不占正式奖池，不写正式中奖表。

## 3. 广告机页面结构

建议广告机抽奖页固定为四区结构：

### 3.1 顶部规则区

展示客户可见规则，不展示后台概率与高价值分配逻辑：

1. 购机 `SN` 是正式抽奖唯一凭证。
2. 无 `SN` 可参与模拟抽奖体验。
3. 扫码填写资料后进入正式抽奖。
4. 中奖结果现场生成并同步后台留档。
5. 客户信息脱敏展示。

### 3.2 左侧参与区

功能：

1. 显示二维码。
2. 显示“正式抽奖”入口状态。
3. 显示“模拟抽奖”入口按钮。
4. 显示当前待抽客户信息卡。
5. 显示 `SN` 校验结果与异常提示。

### 3.3 中间转盘区

功能：

1. 作为主视觉区域。
2. 放置大型转盘与中心抽奖按钮。
3. 支持正式抽奖与模拟抽奖两种状态标识。
4. 提供点击反馈、灯效、粒子、中奖弹出层、音效。

### 3.4 右侧公示区

功能：

1. 显示累计参与人数。
2. 显示当前奖池轮次。
3. 显示当前奖池剩余奖卡数量。
4. 滚动展示最新中奖记录。

公示格式建议：

```text
张* ｜ Y9000P ｜ 抽中智惠四年
李* ｜ 小新 Pro ｜ 抽中智惠三年
王* ｜ 拯救者 R9000P ｜ 抽中智惠五年
```

只显示姓氏加 `*`，不显示完整手机号，不显示完整 `SN`。

## 4. 手机扫码留资页

手机页是正式抽奖资格页，不是转盘主页面。

字段建议：

1. 客户姓名
2. 手机号码
3. 购机机型
4. `SN`
5. 可选备注

提交后状态：

1. 校验通过：写入“待抽奖队列”，广告机显示待抽客户。
2. 校验失败：返回明确原因。

错误提示建议：

1. `SN 不能为空`
2. `该 SN 已参与正式抽奖`
3. `当前活动已结束或未开始`
4. `SN 格式不正确`
5. `系统暂时无法校验，请联系店员`

## 5. 抽奖规则口径

### 5.1 奖池基础结构

每一轮奖池固定 20 张：

1. 智惠三年：15 张
2. 智惠四年：3 张
3. 智惠五年：2 张

### 5.2 切池规则

当前轮次奖池未抽完时：

1. 不允许切换下一池
2. 不允许重置剩余卡数
3. 不允许人工跳过剩余卡

只有当前轮次剩余卡数为 `0` 时，后台才允许开启下一轮奖池。

### 5.3 正式抽奖资格规则

1. 正式抽奖必须有 `SN`
2. 同一 `SN` 只能正式参与一次
3. 无 `SN` 只能走模拟抽奖
4. 可匹配到库存/订单时，建立订单关联
5. 匹配不到库存/订单，但 `SN` 合规时，仍允许正式抽奖
6. 若属于无法匹配库存的 `SN`，则不进入库存关联同步链

### 5.4 模拟抽奖规则

1. 不要求 `SN`
2. 不占正式奖池
3. 不写正式中奖记录
4. 可单独记录互动日志与参与人数

## 6. 后台奖励分配逻辑边界

客户前端不展示以下逻辑：

1. 高价机型优先触发高价值卡的分配策略
2. 一万元以上机型的高价值服务卡倾斜规则
3. 每 3 个高价购机客户触发一次高价值延保卡的内部策略

这些逻辑只存在于后台抽奖引擎中，用于控制正式奖池分配，不在广告机或手机页公开展示。

## 7. 数据同步链路

### 7.1 正式抽奖写入目标

正式抽奖结果需要同步到：

1. SQL 数据库
2. 后台管理平台
3. 广告机前端公示区
4. 抽奖记录查询接口

### 7.2 订单与 `SN` 匹配

匹配顺序建议：

1. 先按 `SN` 匹配本地 SQL 中的序列号主链
2. 再匹配零售订单 / 出库记录
3. 若找到关联订单，则记录 `order_id / order_no / sku / product_model`
4. 若找不到订单，但 `SN` 通过格式和活动资格校验，允许继续抽奖

### 7.3 不匹配库存的处理

对 `PO` 机、外采机、库存外 `SN` 的处理：

1. 允许正式抽奖
2. 保留抽奖记录
3. 不同步库存主链
4. 不强制写订单关联字段
5. 需在后台标识为 `external_sn` 或 `unmatched_sn`

## 9. 接口建议

### 9.1 手机页提交资格

`POST /api/ad-machine/lottery/participants`

用途：

1. 提交客户资料
2. 校验 `SN`
3. 生成待抽资格

### 9.2 正式抽奖

`POST /api/ad-machine/lottery/formal-draw`

用途：

1. 从待抽资格进入正式抽奖
2. 锁定当前轮奖池
3. 消耗一张奖卡
4. 回写正式结果

### 9.3 模拟抽奖

`POST /api/ad-machine/lottery/mock-draw`

用途：

1. 生成模拟结果
2. 不消耗正式奖池

### 9.4 广告机公示数据

`GET /api/ad-machine/lottery/dashboard`

返回：

1. 当前活动状态
2. 当前轮次
3. 剩余奖卡数
4. 累计参与人数
5. 最新中奖滚动列表
6. 当前待抽客户

### 9.5 后台管理端

`GET /api/admin/lottery/campaigns`
`GET /api/admin/lottery/rounds`
`GET /api/admin/lottery/records`
`POST /api/admin/lottery/rounds/open-next`
`POST /api/admin/lottery/records/void`

## 11. 广告机交互细节建议

### 11.1 正式抽奖按钮

需要强化：

1. 大尺寸中心按钮
2. 按下缩放反馈
3. 边缘发光
4. 抽奖中旋转动画
5. 停止时中奖闪光
6. 音效反馈

### 11.2 公示区滚动

建议：

1. 自动滚动最新 20 条
2. 每 3-4 秒切换一条
3. 字段顺序固定：`姓氏* + 机型 + 中奖结果`

### 11.3 异常提示

广告机前台不要显示技术错误堆栈，只显示业务提示：

1. `该 SN 已参与正式抽奖`
2. `当前无可用奖池`
3. `网络异常，请稍后重试`
4. `请先完成扫码留资`

## 12. 风险与未覆盖细节

### 12.1 需要补明确的业务规则

1. `SN` 格式校验规则尚未明确
2. 抽奖活动的开始/结束时间尚未给出
3. 不同门店是否共享同一奖池尚未明确
4. 后台管理员的权限边界尚未明确
5. 抽奖作废后是否退回奖池尚未明确

### 12.2 技术风险

1. 广告机断网时，正式抽奖与 SQL 同步无法保证
2. 手机页提交后若广告机未刷新，待抽状态可能延迟
3. 奖池扣减与结果写入必须事务化，否则会出现“中奖成功但奖池未扣减”
4. 同一 `SN` 并发提交需要唯一约束与幂等保护
5. 外采 `SN` 无库存映射时，后台统计要与正式库存数据明确隔离

### 12.3 建议补充的功能

1. 后台“手动补录正式抽奖记录”能力
2. 后台“作废抽奖记录”审批能力
3. 抽奖操作审计日志
4. 奖池轮次切换审批确认
5. 二维码页面超时失效机制

## 13. 推荐实施顺序

### 第一阶段：方案确认

1. 确认页面结构
2. 确认奖池轮次规则
3. 确认 `SN` 与订单匹配边界
4. 确认后台管理权限

### 第二阶段：数据链路

1. 建 SQL 表
2. 建后台管理 API
3. 建广告机 dashboard API
4. 建手机扫码提交流程

### 第三阶段：前端交互

1. 广告机转盘主页面
2. 手机扫码页
3. 公示滚动区
4. 模拟抽奖链路

### 第四阶段：联调收口

1. 联调 SQL 写入
2. 联调后台管理端
3. 联调广告机展示
4. 验证切池规则
5. 验证 `SN` 去重与异常处理

## 14. 本轮建议结论

当前不建议直接开始写全部代码。  
先以本方案为审阅基准，优先确认以下 5 项：

1. 正式抽奖是否固定为同一 `SN` 只能一次
2. 多门店是否共用同一活动与奖池
3. 作废抽奖记录后是否退回奖池
4. 一万元以上机型的高价值卡倾斜规则是否仅后台生效且不公示
5. 外采/PO 机 `SN` 是否允许保留正式中奖记录但不做库存关联

以上 5 项确认后，再进入代码实施阶段最稳妥。
