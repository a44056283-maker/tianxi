# 报价采集实操校准规则

更新时间：2026-05-15

## 目标

把微信报价采集从“靠临场点击”收成可复用的稳定流程，先看屏幕状态，再执行下一步动作。

## 强制规则

1. 每次微信报价采集，先看屏幕，再点击。
2. “看屏幕”不是主观判断，优先使用：
   - `scripts/classify_wechat_quote_state.py`
   - `scripts/capture_wechat_window.sh`
   - `scripts/capture_and_ocr_wechat_quote.sh`
3. 只有状态明确后，才允许下一次点击。
4. 如果状态是 `unknown`，禁止盲点，必须先复核当前窗口。
5. 禁止口头推测、禁止未实操先报结果。
6. 未完成实际点击、截图、OCR、落盘，不得汇报“已命中”“命中率多少”“已采集完成”。
7. 命中率只能根据实际练习记录计算，不允许凭感觉估算。

## 练习标准

群报价采集、公众号报价采集各自都必须完成至少 `5` 次完整练习。

一次完整练习的定义：

1. 命中正确入口
2. 命中正确内容页
3. 成功截图
4. 成功 OCR
5. 能明确判断：
   - 已拿到真实价格
   - 或被掩码 / 风控 / 登录阻塞

只有这 `5` 步都闭环，才算 `1` 次有效练习。

## 命中率统计口径

1. 命中率只从练习记录台账计算：
   - `docs/报价采集练习记录.md`
2. 统计维度固定为：
   - 入口命中率
   - 内容页命中率
   - OCR 成功率
   - 真实价格命中率
3. 真实价格命中率必须以“拿到未掩码、可写入的真实价格”为准。
4. 掩码价、登录页、错误会话、错误文章都算未命中，不得美化。

## 当前固定顺序

### 公众号报价

1. `classify_wechat_quote_state.py`
2. 确认当前是 Chrome `https://localhost:3001/` 网页微信，不是微信桌面版
3. 再次 `classify_wechat_quote_state.py`
4. 从左上角微信图标进入入口区
5. 再次 `classify_wechat_quote_state.py`
6. 单步点击收藏/方块入口，进入收藏夹
7. 再次 `classify_wechat_quote_state.py`
8. 在收藏夹内打开已经收藏的公众号报价信息，不搜索公众号、不从小程序入口进入、不从桌面微信进入
9. 再次 `classify_wechat_quote_state.py`
10. 若是 `official_article` 或当天报价截图：
   - `capture_and_ocr_wechat_quote.sh`
   - 必要时 `capture_wechat_article_pages.sh`
11. 如果检测到 `maskedPriceDetected = true`
   - 记录阻塞，不写正式价格快照

### 群报价

1. 确认当前是 Chrome `https://localhost:3001/` 网页微信，不是微信桌面版
2. 先命中目标群会话、群文件入口或聊天记录
3. 先截图和 OCR，确认当前是否是：
   - 群文件
   - 当日报价图片
   - 当日报价 Excel
4. 如果网页微信已经把当天 Excel 落到 Selkies 映射目录，可以直接消费该当天文件，但必须报告来源为网页微信映射目录
5. 状态确认后再点下载或打开
6. 下载后先保留原文件，再解析

## 当前原则

1. 命中率提升优先靠：
   - 屏幕状态识别
   - 窗口级截图
   - OCR 验证
   - 最少点击
2. 不是靠增加盲点次数。
3. 外部工具和脚本可以继续加，但必须服务于“先看再点”的流程。
