# 15 Subagent Execution Playbook

更新时间：2026-05-13

## 目的

- 把现有子代理变成固定工位，而不是临时想到什么就派什么。
- 保证每天定时任务到点后，子代理先完成“核查、整理、判定”，主线程再去做真实页面采集和主仓写入。
- 以后所有子代理和外部分发动作，先遵守：
  - `docs/ai-context/20_TASK_DECOMPOSITION_AND_MODEL_ROUTING.md`
  - `docs/ai-context/22_STANDARD_TASK_CARDS_AND_DISPATCH_TABLE.md`

## 固定工位

### Hilbert

- 工位：智店通库存 / SN 一致性核查
- 只做：
  - 对比当天 `latest-inventory-movements.json`
  - 对比 `latest-serial-overrides.json`
  - 对比库存快照里当天出入库影响
  - 找出“数量已变、SN 仍在库中”的冲突
- 不做：
  - 不打开真实业务页面
  - 不直接改主快照

### Ramanujan

- 工位：前端展示与保修区审计
- 只做：
  - 审计 `库存零售专区` 的 SN / 保修展示是否符合当前数据
  - 审计右侧官网链接、滚动区域、显示字段
  - 如需极小 UI 修复，只改前端展示文件
- 不做：
  - 不改采集结果
  - 不冒充保修已采完

### Tesla

- 工位：定时任务时间表 / 收口状态表
- 只做：
  - 整理当天每个时间段的任务状态
  - 标出“已真实完成 / 已执行未收口 / 缺原始输入 / 页面阻塞”
  - 生成主线程下一步顺序
- 不做：
  - 不改业务快照

### Aquinas

- 工位：价格输入链 / MiniMax 归纳链
- 只做：
  - 核查分销报价、灰渠原文、价格快照是否一致
  - 核查 `MiniMax` 只做结构化提取和归纳时还能补哪些环节
  - 产出明确的“可自动归纳 / 不能自动归纳”边界
- 不做：
  - 不让 `MiniMax` 直接替代京东老浏览器会话
  - 不让 `MiniMax` 直接替代智店通页面采集

## 每次下发必须包含

1. 本轮只做什么
2. 本轮不做什么
3. 输入文件或输入来源
4. 成功标准
5. 输出路径或汇报格式
6. 任务类别与最终收口者

## 下发后的执行产物门禁

- 子智能体和定时任务智能体接到任务后，不能只做状态浏览。
- “打开过网页”不算完成。页面类任务必须同时交付：
  - 页面内容复核证据
  - SQL/API/快照写入证据
  - 前端可见复核证据
- 每轮必须至少交付以下三类产物之一：
  1. 新的正式任务报告
  2. 新的手工批次、截图、网页证据或结构化证据文件
  3. 新的阻塞证据文件，并说明卡在什么页面、什么输入、什么登录态或什么字段
- 如果没有新增产物，必须明确写 `execution_failed_noop`，不得写“已执行”“已同步”“已完成”。
- 对价格、链接、竞品、国补类任务，若页面有规格、颜色、套餐、版本、配置区或页面下方同系列规格，必须点击或切换到目标 SKU 规格，并记录选中态；没有点规格不得判定“无同配”。
- 对京东 / 官旗价格复核，输出必须列出 `平台主标题 / 小字副标题或配置副标题 / 已选规格文案 / 商品编号 / preSubsidyPrice / couponAdjustedPrice / postSubsidyPrice`。只采主标题或只报一个最终价格，不算完成。
- 若目标 SKU 存在白色/冰魄白/月幕白等白色变体，子智能体必须额外执行一次白色复采核对，确认白色规格标题、副标题、已选规格、商品编号与价格都对应白色目标 SKU；未完成该复采核对，子智能体只能回报 `executed_not_closed`。
- 页面公式类似 `10999 = 13499 - 1000 - 1500` 时，必须指出主价是 `13499`，券后中间价和国补后价只作备注。
- 对播报和快照重建类任务，脚本成功只代表本地命令成功；必须核对数据源和前端对应子书签。没有前端可见审计时，只能写 `executed_not_closed`。
- 页面类任务必须显式记录 `页面身份 / 已选规格或颜色 / 可见价格或状态区 / 页面时间或群时间线`。缺其中任一项，不得写 `real_completed`。

## 模型档位与并行规则

- 以后只要用户一次下达多任务，默认先拆工位并行执行，不再把所有支线塞进主线程串行处理。
- 一级任务拆解权固定保留给 `Codex`；`GPT` 与 `OpenClaw` 默认只接窄任务执行单。
- 固定模型档位：
  - `S档 / gpt-5.5`：只给高风险、跨模块、强推理、真实业务口径容易误判的复杂任务；不作为日常定时任务常驻模型。
  - `A档 / gpt-5.4`：给智店通、京东/联想价格复核、竞品详情核验、网页微信/灰渠可见证据判断这类中高复杂任务。
  - `B档 / gpt-5.4-mini`：给看门狗、快照重建、日报/播报、开发计划、陈旧库存、纯报告审计这类低风险任务。
- 固定分派原则：
  - 阻塞主线程下一步的关键路径任务，由主线程自己做，或者只在极清晰边界下分给高档模型子智能体。
  - 不阻塞主线的并行审计、日志整理、报告归纳、配置核查，优先分给 `A档/B档` 子智能体。
  - 子智能体不得彼此重叠写同一组文件；如需改文件，必须先明确文件所有权。
- 固定工位与推荐档位：
  - `Hilbert`：库存/SN 一致性核查，默认 `A档`
  - `Ramanujan`：前端展示/保修区审计，默认 `A档`
  - `Tesla`：定时任务状态表/收口状态表，默认 `B档`
  - `Aquinas`：价格输入链、模型与自动化配置边界审计，默认 `A档`，遇到复杂模型路由或高风险策略再升 `S档`
- 子智能体输出只能是：
  - 只读审计结论
  - 指定范围内的窄改动
  - 主线程下一步排序建议
  不得把只读核查写成业务已完成或任务已收口。

## 新增固定并行分工（2026-05-24 起）

- 以后遇到“规则落地 + 定时任务审计 + SQL主链整改 + 备份链路整改”这类组合任务，默认拆成：
  - `Tesla / B档`：定时任务、watchdog、收口状态、缺证据门禁审计
  - `Hilbert / A档`：库存、SN、入库/出库 SQL 主链残留点审计
  - `Aquinas / A档`：价格、链接、活动、模型与配置边界审计
  - `Ramanujan / A档`：前端映射是否仍有静态占位、是否已按 SQL/API 显示
- 主线程保留：
  - 真正的 SQL 写入改造
  - 真正的启动链/备份脚本改造
  - 真正的文档规则生效与最终集成
- 多智能体之间禁止重叠写同一组文件；如必须改文件，先按所有权拆开，例如：
  - API/SQL 文件归主线程或单独 worker
  - 文档规则归主线程
  - 子智能体默认只读

## 主线程与子代理边界

### 外部采集硬门禁

- 主线程和所有子代理都只能使用当前已登录的默认 Chrome `user` 会话处理智店通、网页微信、京东、联想官旗、天猫等外部登录态采集。
- 禁止打开新的浏览器、空白浏览器、新 Chrome Profile，禁止清理登录缓存，禁止退出账号后重登。
- 禁止使用 Browser、in-app browser、browser-use、Playwright、Puppeteer、Chromium launch 打开或推进外部采集页面。
- 子代理如无法复用老 Chrome 会话，必须输出 `blocked_page_risk` 或 `executed_not_closed`，不得换工具绕过。
- 教育补代扫必须从最新消息进入图片查看器，再按历史方向自底向上完整采到本轮目标边界；进入群后打开看到的第一张相关照片，优先用图片查看器左箭头或键盘左键逐张向历史图片方向查看，只按照片层推进，不下拉聊天；采集后必须写 SQL、刷新前端并检查前端可见；未到边界、未打开图片、只看到上传/核销卡、未写 SQL 或未前端复核时，不得判断完成。
- 教育补代扫自 2026-06-04 起统一改为“产品信息图 + 核销码图”主采集标准，两个群完全同规则：
  - 必须从最新位置进入图片查看器，再按历史方向自底向上完整回扫到最早目标边界
  - 不再以箱码图作为主采集标准；箱码只作辅助证据
  - 普通单扫至少采 `1` 个产品信息 + `1` 个核销码
  - 二件套至少采 `2` 个产品信息 + `3` 个核销码
  - 三件套至少采 `3` 个产品信息 + `4` 个核销码
  - 只要客户电话一致、SN一致、姓名一致三者任一成立，即可认定为同一订单
  - 未完整回扫到本轮目标边界、未持久化订单与客户字段、未写 SQL 或未前端复核时，不得判断完成
- 教育补代扫套装归类必须执行新硬规则：只有命中营销库 MTM 物料号且命中同一真实销售出库单号，才允许归到三件套、两件套或拯救者双屏畅玩两件套；套装费用按整单只计算一次，不得再叠加单品代扫费；未匹配成功的历史候选记录只能标成待匹配并持续持久化，不得冒充已结算套装。
- 只要命中三件套、两件套或拯救者双屏畅玩两件套，记录就绝不能再回落到单品代扫；单扫页只允许保留未命中套装规则的真实单扫记录。

### 主线程必做

- 智店通真实页面采集
- 老的已登录京东浏览器实时核价
- 正式写入：
  - 价格快照
  - 库存流水
  - SN 覆盖层
  - 最终前端快照

### 子代理必不做

- 不把只读核查当成真实同步
- 不把自动报告当成今日数据完成
- 不把旧文件沿用当成今日新采集

## 今日执行口径

1. 库存、出库、SN 同步优先级高于保修。
2. 实时进货价输入链优先级高于 MiniMax 试验。
3. 保修采集只在主线程空闲时继续跑。
4. `MiniMax` 当前定位为：
   - 页面结果结构化
   - 证据归纳
   - 批量待办整理
5. `MiniMax` 当前不是：
   - 京东采价浏览器替身
   - 智店通页面采集替身
