# 17 Scheduled Task First Principles Standard

更新时间：2026-05-25

## 第一性原理

定时任务的目标不是“触发过”，而是把业务事实从真实输入推进到可复核产物。

最小闭环固定为：

1. 输入：确认今天真实输入是否存在。
2. 动作：按任务类型执行采集、导入、解析、写回或播报。
3. 产物：生成正式报告、手工批次/证据、或阻塞证据。
4. 同步：需要写回的任务必须更新 SQL/API/快照和前端数据。
5. 验收：检查任务报告、关键指标和前端可见字段。
6. 状态：只能写 `real_completed`、`executed_not_closed`、`blocked_missing_input`、`blocked_page_risk`、`execution_failed_noop`。

## 执行规则

- 没有新增产物，状态就是 `execution_failed_noop`。
- 脚本成功不等于业务完成；前端/API/SQL 不一致时是 `executed_not_closed`。
- 缺当天输入时是 `blocked_missing_input`，旧文件只能作为历史沿用证据。
- 页面登录、白屏、403、验证码、滑块、安全验证时是 `blocked_page_risk`。
- 外部网页证据必须使用当前 Chrome 登录态低频可见操作。
- 每个定时任务在每个站点只允许保留 `1` 个稳定工作页作为采集入口；不得在同一站点长期并开多个采集页。需要临时打开详情页、图片页或候选页时，记录证据后立即关闭并回到该站点唯一保留的工作页。
- 固定站点保留口径：
  - 网页微信：只保留一个 `https://localhost:3001/` 主页。
  - 智店通：只保留一个 `https://retail-pos.lenovo.com/` 业务页。
  - 京东：每轮只保留一个京东工作页，详情页核验后立即关闭。
  - 联想官网：每轮只保留一个联想商城工作页，详情页核验后立即关闭。
  - 联想保修：每轮只保留一个联想保修查询页，逐条低频查询，不并开多个 SN 结果页。
- 若同一站点已经存在多个旧标签页，当前轮必须先收敛到唯一工作页，再继续采集；否则本轮只能写 `executed_not_closed`，不能写完成。
- 浏览器采集类任务在真正采集前，必须先做一次“标签页数量前置检查”：统计本轮涉及站点的相关标签页数量，执行收敛，再记录收敛后数量。汇报里必须至少带出 `tabPrecheckBefore` 与 `tabPrecheckAfter`，否则视为证据不完整。
- 所有定时任务无论本轮实际采集、导入、审计或播报了多少条数据，结束前都必须把 Chrome 恢复到固定 `5` 个页面：`WeChat Selkies / https://localhost:3001/`、`店铺列表 - 智慧零售云平台 / https://retail-pos.lenovo.com/`、`联想智慧零售系统 / http://127.0.0.1:5174/`、`联想官网`、`京东`。未恢复到这 `5` 页，本轮只能写 `executed_not_closed`。
- 任何智能体不得私自退出、停用、暂停、删除、覆盖、降级或改为非 `ACTIVE` 的既有定时任务。若任务未收口，只能保留自动化继续运行，并把状态如实写为 `executed_not_closed` 或 `blocked_*`。
- 若发现自动化列表掉线、任务被外部操作挪动、任务状态异常、或本轮未完成，必须立即反馈飞书群，写清任务名、当前状态、未完成原因、阻塞点、待人工动作与恢复结果；禁止静默结束。
- 定时任务恢复、重启或重新加载后，禁止批量补跑过去未完成的历史任务。自动执行只允许处理当前时间窗到期的当前轮任务；历史 `executed_not_closed / blocked_* / missed` 只保留报告、阻塞原因和飞书提醒，不再自动排队补跑。
- 同一个计划时段只允许自动尝试 `1` 次。该时段一旦已经执行过，无论结果是完成还是失败，都不得在同一时段内反复重试，更不得和后续时段任务堆叠并发。
- 任何智能体采集到的数据任务结果，必须先同步写入 SQL 数据库或受控 SQL 证据表，再触发 API/快照/映射刷新，并且必须亲自打开前端对应页面校验 UI 已显示本轮最新数据，才允许写 `real_completed`。
- 如果只完成采集、只生成 JSON、只写本地文件、只更新日志、只更新 SQL 但前端未刷新、或前端刷新后 UI 仍未显示本轮关键字段，统一只能写 `executed_not_closed`。
- 汇报必须写清 SQL 写入结果、前端 URL、子书签、亲眼看到的关键字段和同步时间；没有这些验收字段，视为未收口。
- 价格/链接/竞品复核必须点击或切换规格并记录选中态；未点击规格不得判定“无同配”。
- 价格/链接/竞品复核必须结构化记录 `平台主标题 + 小字副标题/配置副标题 + 已选规格文案 + 商品编号`。只采到主标题、没有副标题或已选规格字段时，证据不完整，只能写 `executed_not_closed`。
- 价格公式必须拆成 `preSubsidyPrice / couponAdjustedPrice / postSubsidyPrice`。门店主参考价固定取营销、教育、PO、国补前原始正常价；券后中间价和国补后红字到手价只作备注。示例 `10999 = 13499 - 1000 - 1500` 时主价必须写 `13499`。
- 京东 / 联想官旗任一来源若当前目标规格显示 `无货 / 到货通知 / 售罄 / 暂不销售 / 待发布`，该来源只保留证据和链接，不得继续压低主参考价；主参考价必须降级切到另一个“有价且可售”的来源。
- 白色、黑色、灰色等颜色差异必须视为有效配置差异。游戏本白色机型不得直接沿用黑色价格；只有目标颜色被真实选中且标题、规格、价格同页一致时，才允许写回。
- 超过 `36` 小时未复核的平台价只能保留为历史证据，不得继续参与当前门店主参考价计算。
- 白色复采核对为强制门禁：凡涉及价格、链接、竞品、播报、审计、快照重建或前端核对，只要 SKU 或页面存在白色/冰魄白/月幕白等白色变体，必须额外做一次白色复采核对，确认已切到白色规格，且标题、副标题、已选规格、商品编号、主价口径全部对应白色目标 SKU；未完成白色复采核对，本轮只能写 `executed_not_closed`。

## 标准输出

每轮只汇报以下字段：

- `taskName`
- `status`
- `newArtifacts`
- `keyMetrics`
- `frontendChecked`
- `blockingReason`
- `nextAction`

禁止输出泛泛计划；没有执行就明确写失败或阻塞。
