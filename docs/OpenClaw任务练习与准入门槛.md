# OpenClaw 任务练习与准入门槛

更新时间：2026-05-17

## 目的

- 先把 OpenClaw 的任务顺序、技能命中、页面恢复路径、回执输出跑通。
- 没有练习通过的任务，不得直接交给定时任务。
- 防止出现“任务已创建，但实际一运行就卡住”的假交接。

## 总原则

1. 先练习，再定时。
2. 先单任务跑通，再扩到同类任务。
3. 先人工可见练习，再允许自动编排。
4. 未达到准入门槛，只能算“练习中”，不能算“已交接”。

## 固定准入门槛

每个要交接给 OpenClaw 的任务，必须同时满足：

1. 同一任务连续练习 `5` 遍。
2. `5` 遍里每一遍都必须：
   - 正确进入目标页面
   - 正确识别当前页面状态
   - 不乱点、不连点、不新开空白窗口
   - 能输出真实摘要
   - 能写 receipt 或聊天回执
3. 至少 `3` 遍能完整走通“任务下发 -> 执行 -> 回话回流 -> Codex 看板可见”。
4. 任意一遍出现下面情况，连续计数清零，重新开始：
   - 页面卡死
   - 白屏
   - 持续转圈
   - 打开错误页面
   - 回执未写出
   - 聊天板未收到真实回复
   - 数据口径错误

## 练习阶段

### 阶段1：投递链路练习

目标：

- Codex 下发一条任务后，OpenClaw 必须真实收到。
- OpenClaw 完成后，聊天板必须真实看到回复。

通过标准：

1. 会话收到任务文本。
2. OpenClaw 产生回复。
3. 首页聊天板自动回读该回复。

### 阶段2：页面命中练习

目标：

- OpenClaw 能按既定入口进入正确页面。
- 每一步先看状态再点。

通过标准：

1. 连续 5 遍命中目标入口。
2. 不出现盲点和连点。
3. 能区分：
   - 正常页
   - 转圈页
   - 白屏页
   - 403
   - 登录页
4. 智店通类页面进入后转圈超过 `5` 秒，必须立即返回首页稳定入口。
5. 智店通练习只接受人工单击路径，不接受脚本推进路径。

### 阶段3：证据与回执练习

目标：

- 每轮练习都要留下真实证据。

通过标准：

1. receipt 写出成功。
2. 路径真实存在。
3. 摘要能被 Codex 读取和复核。

### 阶段4：任务顺序练习

目标：

- 把单任务内部顺序跑顺。

例如：

- 智店通同步：
  - 采购入库 -> 销售出库 -> 其他出库
- 报价采集：
  - 进群/进公众号 -> 找当天报价 -> 截图/原文 -> 结构化摘要 -> 回执

通过标准：

1. 顺序不乱。
2. 中间卡住时能正确阻塞退出。
3. 不跳步、不补写假完成。
4. 智店通任务中任何一页转圈超过 `5` 秒，都要执行“返回首页 -> 重进”的恢复顺序。

## 当前正式规则

从 2026-05-17 起：

1. OpenClaw 只允许进入“练习模式”。
2. 未完成 5 轮练习通过前，不继续批量交接定时任务。
3. Codex 的工作变成：
   - 监督练习
   - 校对每一轮结果
   - 记录阻塞点
   - 修提示词与顺序
4. OpenClaw 练习通过后，才允许：
   - 固化成正式任务
   - 再进入定时任务

## 能力训练拆解

不要一上来就训练“完整采集任务”，而要先按能力拆开训练。

每类业务任务都必须先拆成下面 6 个基础能力：

1. 进页
2. 找入口
3. 导出或截图
4. 确认文件落点
5. 提取内容
6. 分类归档并回执

只有前一项通过，才允许练下一项。

### 能力1：页面进入训练

目标：

- 能进入正确页面。
- 能区分正常页、白屏页、转圈页、登录页。

通过标准：

1. 连续 `5` 轮进入同一目标页。
2. 每轮都能说清：
   - 当前页面名称
   - URL 或可见标题
   - 当前状态
3. 智店通页面转圈超过 `5` 秒立即退出，不死等。

### 能力2：入口定位训练

目标：

- 能在正确页面中命中正确按钮、菜单、页签、日期框、导出按钮。

通过标准：

1. 连续 `5` 轮正确命中同一入口。
2. 不允许把相似入口混淆：
   - `调拨出库` 不能当 `销售出库`
   - `其他出库` 不能当 `销售日报`
3. 点击失败时，能明确说明是：
   - 坐标偏差
   - 无障碍树缺失
   - 页面控件未初始化
   - 页面转圈

### 能力3：导出动作训练

目标：

- 能完成导出、下载、截图、保存原始证据。

通过标准：

1. 能说清当前任务是：
   - 导出 Excel
   - 下载原图
   - 截图保存
   - 只读核查无需导出
2. 导出后必须确认：
   - 是否真的弹出下载
   - 是否真的生成文件
   - 文件名大致是什么
3. 如果没有导出能力，必须明确写：
   - “本轮只能截图，不能导出”
   - “本轮只能定位导出按钮，不能确认落盘”

### 能力4：文件落点确认训练

目标：

- 能确认导出文件到底保存到了哪里。

通过标准：

1. 每轮都要写出：
   - 文件路径
   - 文件名
   - 时间
2. 如果无法确认落点，任务不能算完成。
3. 默认优先确认：
   - 浏览器下载目录
   - 项目 `artifacts/manual/openclaw/`
   - 用户手动指定目录

### 能力5：内容提取训练

目标：

- 能从导出的文件、截图、页面文本中提取出结构化结果。

通过标准：

1. 能明确当前提取来源：
   - 页面可见文本
   - 截图 OCR
   - Excel 导出
   - 图片原文
2. 能输出最小结构化字段。
3. 提取不出来时不能瞎猜，必须写：
   - OCR 失败
   - 表格列无法识别
   - 页面字段不可见

### 能力6：分类归档训练

目标：

- 能把提取结果按业务分类归档，而不是堆成一团原始文本。

通过标准：

1. 能明确这次归到哪个类别：
   - 智店通同步证据
   - 群报价
   - 公众号报价
   - 京东零售价
   - 竞品监控
2. 能写出本轮分类键：
   - 日期
   - 来源
   - 产品类别
   - 产品系列
   - 产品型号
3. 能写 receipt。

## 业务训练顺序

### A. 智店通类

固定训练顺序：

1. 进首页
2. 进目标菜单
3. 找日期框 / 筛选框 / 导出按钮
4. 执行导出或确认无法导出
5. 确认文件落点
6. 提取导出内容
7. 分类为：
   - 采购入库
   - 销售订单证据
   - 其他出库
   - 销售日报

### B. 群报价 / 公众号报价类

固定训练顺序：

1. 进入来源页面
2. 找当天有效报价
3. 截图或保存原文
4. 确认原始文件路径
5. OCR / 文本提取
6. 按：
   - 产品分类
   - 产品系列
   - 产品型号
   分类

### C. 京东 / 联想 / 竞品类

固定训练顺序：

1. 打开已锁定详情页
2. 确认商品是否一致
3. 截图价格与链接
4. 保存证据
5. 提取：
   - 标题
   - 配置
   - 价格
   - 国补说明
6. 按品牌 / 品类 / 型号归档

## 督导提示词模板

Codex 给 OpenClaw 下发任务时，必须写清 6 件事：

1. 这轮只做什么
2. 这轮不做什么
3. 成功标准是什么
4. 导出后要确认什么
5. 结果要按什么分类
6. 做不了时要如何阻塞退出

固定模板：

```text
你这轮不是“完成整个业务”，而是只训练一个能力环节。

本轮只做：
1. ...
2. ...

本轮不做：
1. 不写 latest-*.json
2. 不补做其他页面
3. 不用 CDP/browser/脚本（若本轮禁用）

成功标准：
1. ...
2. ...

如果涉及导出，必须额外确认：
1. 是否真的触发导出
2. 文件名是什么
3. 保存路径是什么

如果涉及提取，必须额外输出：
1. 来源类型（页面/截图/OCR/Excel）
2. 提取出的关键字段
3. 分类结果

如果做不到，不准硬做，直接写：
1. 卡在哪一步
2. 为什么做不到
3. 下一轮应该单独训练哪一个能力
```

## 练习日志要求

每轮至少记录：

1. 任务名
2. 第几轮
3. 是否命中目标页
4. 是否正确识别页面状态
5. 是否正确输出回执
6. 是否成功回流聊天板
7. 是否存在阻塞
8. 阻塞原因
9. 下轮修正点
