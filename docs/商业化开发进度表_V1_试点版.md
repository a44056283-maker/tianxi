# 联想智慧零售系统商业化开发进度表 V1.0

更新时间：2026-05-24  
适用目标：`70分可卖试点版`  
当前主线程：优先把 `库存`、`出入库`、`营销/价保活动` 三条主链先做成可售卖能力。

## 1. 当前基线

- 当前商业化成熟度基线：`33/100`
- 当前可售卖口径：`可卖试点定制版，不可按成熟标准 SaaS 对外大规模售卖`
- 当前优先模块：
  - `库存 / SN`
  - `出入库同步`
  - `营销 / 价保 / 教育补 / PO`
- 当前阶段目标：先做到 `70/100`，形成 `1-3 家门店可交付试点版`

## 2. 本阶段只做什么

- 把库存、SN、销售出库、采购入库、其他出库切到稳定 SQL 主链。
- 把营销活动、价保、教育补、PO 做成 `当前态 + 历史态 + 审计态` 三层可追溯结构。
- 补齐最小商业化交付能力：
  - 租户/门店基础结构
  - 账号/角色骨架
  - 初始化导入
  - 备份恢复
  - 最小 API 文档
  - 试点版验收清单

## 3. 本阶段明确不做什么

- 不先做全行业通用 ERP。
- 不先做复杂总部经营分析。
- 不先做完整收费授权系统。
- 不先做大而全的开放平台。
- 不把静态快照展示包装成已经商业化完成。

## 5. 三条主链的完成定义

| 主链 | 当前问题 | 试点版完成定义 |
| --- | --- | --- |
| 库存 / SN | 仍存在快照过渡、手工补口径、部分页面静态读链 | 库存、SN、库存台账、库存异常全部从 SQL/API 出；静态 JSON 仅保留镜像缓存 |
| 出入库 | 销售/采购/其他出库与库存真值还没完全统一到商业主链 | 销售出库、采购入库、其他出库、库存流水、SN库存订单能同链追溯，并能做一致性审计 |
| 营销 / 价保 / 活动 | 活动、教育补、PO、价保仍有展示层与存储层混用问题 | 每条活动和价保都有历史冻结、来源留痕、销售日固化、可审计变更记录 |

## 7. 试点版验收口径

- `库存模块`
  - API/SQL 为主事实源
  - 库存、SN、台账、异常口径一致
  - 可做同日重建与追溯
- `出入库模块`
  - 销售、采购、其他出库、库存流水、SN 流转可统一追溯
  - 重跑导入不重复
  - 异常单可标记并审计
- `营销模块`
  - 价保、教育补、PO、营销活动可冻结历史
  - 历史记录不被新快照覆盖
  - 可追来源、追销售日、追变更人
- `交付能力`
  - 可初始化新门店
  - 可完成最小账号权限配置
  - 可备份、可恢复、可归档
  - 有最小 API 文档与试点实施清单
