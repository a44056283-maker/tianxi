# 智店通教程生成计划

更新时间：2026-05-17  
适用范围：`商品 / 订单 / 库存` 三大类中当前真正要用的子菜单  
执行方式：`一次只生成一个教程 -> 用户审计通过 -> 再生成下一个`

## 1. 当前原则

1. 暂时不用的子菜单先放一边，不抢进度。
2. 先做真正会进入每日业务链的页面。
3. 每个页面先手动实操确认，再写教程。
4. 教程写完后，先由你审计。
5. 审计通过后，才允许进入下一个页面教程。

## 2. 当前优先级排序

### 第一阶段：先做核心主链教程

1. `商品 -> 门店商品`
2. `订单 -> 线下门店订单`
3. `库存 -> 商品库存`
4. `库存 -> 库存流水`
5. `库存 -> SN库存订单`
6. `库存 -> 库存订单`

### 第二阶段：补强支撑类教程

7. `商品 -> 经销商商品`
8. `商品 -> 仓库商品`
9. `订单 -> 线下门店退单`
10. `订单 -> 线上订单`
11. `订单 -> 线上退单`
12. `库存 -> 销售成本价维护`

### 第三阶段：低优先级补充教程

13. `商品 -> 收银热卖商品`
14. `商品 -> 供应商管理`
15. `库存 -> 库位库存`
16. `库存 -> 库存总览`
17. `库存 -> 调拨入库`
18. `库存 -> 商品入库`
19. `库存 -> 其他出入库`
20. `库存 -> 同店换库位`

## 3. 每个教程固定模板

每次只生成一个页面教程，结构固定为：

1. 页面名称
2. 页面业务定位
3. 页面路由
4. 进入路径
5. 页面可见区域说明
6. 筛选条件说明
7. 按钮说明
8. 表格字段说明
9. 导出能力说明
10. 导出文件落点确认方式
11. 后续数据提取方式
12. OpenClaw 训练要点
13. 常见误操作和阻塞点

## 4. 审计通过标准

你审计一个教程时，主要看这几件事：

1. 页面路径对不对
2. 页面业务定位对不对
3. 筛选项有没有漏
4. 按钮有没有漏
5. 表格字段有没有写错
6. 是否把不用的能力写成了会用
7. 是否把阻塞页写成了已完成页

只要有一项不对，就先修当前教程，不进入下一个。

## 5. 第一份要生成的教程

第一份固定是：

`商品 -> 门店商品`

原因：

1. 它是门店商品知识库。
2. 后续每日门店售价同步会围绕它展开。
3. 它是商品类里最靠近我们本地系统每日价格更新的一页。

## 6. 当前执行规则

从现在开始：

1. 不并行生成多个页面教程。
2. 不跳着写。
3. 不把第二阶段、第三阶段页面抢到前面。
4. 先把 `门店商品` 教程打磨通过，再继续。
