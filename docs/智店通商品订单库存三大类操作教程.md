# 智店通商品-订单-库存三大类操作教程

更新时间：2026-05-17  
来源：基于已登录 `Google Chrome` 可见页面的手动实操观察  
适用对象：Codex / OpenClaw  
适用范围：只覆盖 `商品`、`订单`、`库存` 三大主菜单下当前真实可见子菜单

## 1. 使用边界

1. 本教程只记录已手动打开并确认过的页面结构。
2. 不再扩展学习 `组织 / 账号 / 设备 / 财务 / 数据`。
3. 后续训练和定时任务，先按本教程理解业务角色，再按页面教程执行。
4. 一个页面先完成：
   - 进页
   - 状态识别
   - 按钮识别
   - 是否可导出
   - 文件是否落盘
   - 内容如何提取和归档
5. 页面能打开，不等于已经具备正式采集能力。

## 3. 业务角色先理解

这一部分不是猜，是为了后续训练和采集不跑偏。

### 3.1 商品类角色

#### 经销商商品

- 路由：`/lenovo/web/product/list-city`
- 页面标题：`创建自有商品 - 智慧零售云平台`
- 实际用途：
  - 联想商品主商品池
  - 商品源头信息浏览
  - 商品发布前上游知识库
- 当前页面特征：
  - 筛选：组织架构、商品分类、商品信息、商品状态、69码、PN/MTM、商品类型
  - 按钮：`搜索`、`批量发布`、`配置税率`
  - 标签：`联想商品 / 自有商品 / 全部`
  - 行操作：`发布商品 / 查看 / 编辑`

#### 门店商品

- 路由：`/lenovo/web/product/list-store`
- 页面标题：`店铺商品 - 智慧零售云平台`
- 业务定位：
  - 这是门店侧商品知识库。
  - 这里承接我们后续每日生成的门店售价。
  - 后续“每日门店售价同步”优先理解为同步到这层商品视图，而不是经销商商品页。
- 当前页面特征：
  - 筛选：门店、商品分类、商品信息、PN/MTM、69码、商品类型、商品状态
  - 按钮：`搜索`、`批量上架`、`批量下架`、`导出`
  - 标签：`联想商品 / 自有商品 / 全部`
  - 表头重点：商品信息、商品来源、商品分类、商品类型、商品编码、状态、销售渠道、更新
  - 行操作：`查看 / 编辑`
- 当前理解：
  - 我们本地系统的每日零售价、活动价、同步结果，后续需要围绕这页的商品体系做映射和训练。

#### 仓库商品

- 路由：`/lenovo/web/product/list-depot`
- 页面标题：`创建自有商品 - 智慧零售云平台`
- 业务定位：
  - 仓库侧商品知识库
  - 偏仓储流转和仓库商品视图，不是门店售价主同步页
- 当前页面特征：
  - 筛选：仓库、商品分类、商品信息、PN/MTM、69码、商品类型、商品状态
  - 按钮：`搜索`、`批量上架`、`导出`
  - 标签：`联想商品 / 自有商品 / 全部`
  - 行操作：`查看`

#### 收银热卖商品

- 路由：`/lenovo/web/product/sellinggoods`
- 页面标题：`收银热卖商品 - 智慧零售云平台`
- 业务定位：
  - 收银端热卖商品快捷区
  - 适合做“热卖推荐位”训练，不是全量商品主档
- 当前页面特征：
  - 筛选：门店
  - 按钮：`搜索`、`选择商品`
  - 表头重点：商品信息、分类、销售渠道、商品编码、是否组合商品、是否上架、操作
  - 行操作：`删除 / 置顶 / 上移 / 下移 / 下移`

#### 供应商管理

- 路由：`/lenovo/web/product/supplier-v2`
- 页面标题：`供应商管理 - 智慧零售云平台`
- 业务定位：
  - 供应商档案页
  - 给采购入库和供应商维度管理用
- 当前页面特征：
  - 筛选：供应商名称、状态
  - 按钮：`搜索`、`新建`

  - 行操作：`禁用 / 删除 / 编辑`

## 4. 当前需要优先做教程和训练的页面

按业务价值排序：

1. `商品 -> 门店商品`
   - 因为它是门店商品知识库，也是后续每日门店售价同步的主要商品视图
2. `订单 -> 线下门店订单`
   - 因为它是门店零售订单主页
3. `库存 -> 商品库存`
   - 因为它是库存真值主页
4. `库存 -> 库存流水`
   - 因为它是库存流转主证据页
5. `库存 -> SN库存订单`
   - 因为它是 SN 级补证主页
6. `库存 -> 库存订单`
   - 因为它是当前控件训练难点
