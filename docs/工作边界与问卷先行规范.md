# 工作边界与问卷先行规范

## 核心原则

任何软件框架、功能模块、数据采集流程、价格写入流程，在动手实现或执行前，必须先通过问卷确认目标、边界、采信口径和验收标准。

未经问卷确认，不允许直接运行会改变业务数据的采集、同步、报价、URL 锁定、库存成本覆盖、售后信息写入等操作。

## 标准流程

1. 生成问卷
   - 先围绕业务目标、数据来源、采信规则、写入范围、异常处理和验收标准提问。
   - 问卷必须使用短问题，方便逐条回答。
   - 对已经明确的内容，要写成默认假设，并让用户确认或纠正。

2. 确认边界
   - 明确哪些动作只读，哪些动作会写入文件、数据库或前端展示数据。
   - 明确哪些来源可以采信，哪些来源只作线索。
   - 明确遇到验证码、登录、风控、价格冲突、型号冲突时的停止规则。

3. 生成操作流程
   - 只有在问卷达到要求后，才生成采集流程、CLI 入口、字段映射和异常处理规范。
   - 采集流程必须分成“锁定 URL/来源”和“刷新实时数据”两个阶段。
   - 未锁定 URL 或未校准匹配规则的数据，不允许进入正式报价引擎。

4. 执行与验证
   - 执行前再次说明将要运行的命令和会写入的文件。
   - 执行后给出采集数量、跳过数量、错误数量、写入文件和样例。
   - 发现错配或口径冲突时，先暂停并更新问卷/规范，再继续。

## 数据采集工作边界

库存采信：
- 智店通库存导出和 SN 导出是库存 SKU、库存数量、成本价和售后队列的基准来源。
- 库存快照可以更新，但必须保留原始导出来源路径和生成时间。

成本价：
- 智店通库存成本为主采信来源。
- 群采集和公众号批发价只作为实时进货成本参考，必须保留原文、截图或可追溯链接。
- 未锁定来源和解析规则前，不得覆盖库存成本。

实时零售价：
- 京东、联想官网、淘宝百亿补贴都必须先锁定指定网址。
- 只对已锁定且校准通过的网址刷新价格。
- 不使用 JustOneAPI 等第三方收费接口。
- 使用 Codex 原生电脑操控或 Chrome 登录态进行类人采集。

售后信息：
- 售后和保修信息按 SN 队列采集。
- 遇到验证码、滑块、短信、登录、风控时立即暂停，等待人工处理。

## 京东采集特别边界

- 京东只使用联想京东自营旗舰店和明确认可的备用店铺/页面。
- 优先从店铺首页、系列菜单、英雄卡进入商品详情页，不使用泛搜索作为主流程。
- 京东详情页必须是 `https://item.jd.com/数字.html` 形式。
- 商品配置必须和库存 SKU 匹配；同系列但 CPU、显卡、内存、硬盘、尺寸、颜色不一致时不得写入。
- 采集活动时必须区分展示价、PLUS 立减、平台满减、国家补贴、教育补贴、企业会员补贴和服务加价。

## 后续所有功能开发适用

以后新增软件框架、页面功能、采集器、CLI、自动化流程，都必须先生成问卷并获得确认。
问卷确认前，只允许做只读分析、计划文档和不改变业务数据的检查。
