# 报价来源存储与保留规则

更新时间：2026-05-15

## 适用范围

适用于报价来源库中的所有微信相关报价来源：

1. 分销群报价
2. 灰渠公众号报价

以及后续在 `报价来源` 下新增的所有子书签。

## 子书签顶部必须展示的信息

每个报价来源子书签顶部，必须固定展示：

1. 本地磁盘原始文件路径
2. 结构化快照路径
3. 最新采集时间
4. 最新报价日期

禁止只展示汇总数，不展示真实落地路径。

## 当前统一路径规则

### 分销群报价

- 原始文件路径：
  - `apps/inventory-sync/artifacts/manual/distributor-quotes/`
- 结构化快照：
  - `apps/inventory-sync/artifacts/latest-distributor-quotes.json`
  - `apps/web-cockpit/public/data/latest-distributor-quotes.json`

### 灰渠公众号报价

- 原始文件路径：
  - `apps/inventory-sync/artifacts/manual/gray-wholesale-YYYY-MM-DD.txt`
  - `apps/inventory-sync/artifacts/manual/gray-wholesale-YYYY-MM-DD.md`
- 结构化快照：
  - `apps/inventory-sync/artifacts/latest-gray-wholesale-quotes.json`
  - `apps/web-cockpit/public/data/latest-gray-wholesale-quotes.json`

## 保留周期

所有原始报价文件至少保留 `30` 天。

在这 `30` 天内：

1. 不删除原始文件
2. 不覆盖原始文件日期
3. 不把后补文件伪装成当天原始证据

## 归档规则

每 `30` 天执行一次报价来源归档：

1. 将超过 `30` 天的原始文件按月份打包
2. 归档包保存到：
   - `apps/inventory-sync/artifacts/archives/quotes/YYYY-MM/`
3. 打包完成后，删除已归档的原始源文件
4. 结构化快照不作为原始证据替代品，原始证据必须先完成打包

## 前端实现要求

后续 `报价来源` 子书签改造时，必须做到：

1. 按产品类别 -> 产品系列 -> 产品型号分类展示
2. 即使门店当前无库存的商品，只要当天报价来源库中存在，也要能检索到
3. 每个类别区块显示自己的：
   - 最新采集时间
   - 最新报价日期
   - 原始文件路径

## 禁止

1. 不展示路径来源的“黑箱报价库”
2. 不保留不足 30 天就删除原始文件
3. 不直接删除源文件而不做打包归档
4. 不把归档包当作当天原始证据
