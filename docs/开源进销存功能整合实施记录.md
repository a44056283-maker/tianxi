# 开源进销存功能整合实施记录

更新时间：2026-05-12

## 1. 本次整合原则

本项目继续以现有 `FastAPI + React + 智店通同步器` 为主系统，不直接迁入大型开源 ERP/POS。

本次吸收的是开源系统的成熟业务模型和流程边界：

- InvenTree：库存、SKU、SN、库存流水、API-first、插件式外部集成。
- ERPNext：销售单、采购单、客户、供应商、进销存闭环。
- Open Source Point of Sale：门店收银、销售出库、收货、退货、权限和报表流程。
- OpenEPaperLink / Odoo SES-imagotag 模块：电子价签设备、模板、商品绑定、更新任务。

代码实现保持在本项目内自研，避免直接绑定不同技术栈和许可证。

## 4. 当前行为

`GET /api/retail-core/status` 会：

- 初始化 SQLite 表结构。
- 从 `latest-adjusted-inventory-snapshot.json` 种子导入完整 SKU 和 SN 基础数据。
- 从 `latest-serial-overrides.json` 合并 SN 入库时间、供应商、库位、入库单等覆盖信息。
- 从现有库存快照同步分类映射：智慧零售展示分类、智店通原始子类别、京东/联想导购子类。
- 从 `latest-inventory-movements.json` 同步已有采购入库、销售出库、非零售出库流水。
- 初始化员工、智店通、联想保修、电子价签网关三个外部系统占位。

`GET /api/retail-core/category-tree` 会：

- 返回 `product_category_node` 分类节点。
- 返回 `sku_category_mapping` SKU 分类对照。
- 保留智店通 `sourceCategory`，保证后续同步回智店通时子类别口径一致。

`POST /api/sales/orders` 会：

- 写入本地 JSON 销售台账。
- 写入 SQLite `sales_order`、`sales_order_line`。
- 写入 SQLite `inventory_movement`，类型为 `sales_outbound`。
- 写入 `latest-inventory-movements.json`，保持现有前端兼容。
- 创建 `push_sale_to_zhidiantong` 同步任务。

`POST /api/price-tags/update-tasks` 会：

- 创建 `price_tag_update_task`。
- 创建 `push_price_tag_update` 同步任务。
- 后续可对接汉朔、SOLUM、SES-imagotag 或其他电子价签网关。
