# 教育补代扫箱码补扫任务

任务目标：在当前已登录 Chrome 网页微信中，补齐智店通入库群当天教育补代扫箱码扫描清单，并把结果落盘为正式证据或阻塞回执。

## 固定入口

- 只能使用用户当前已经登录的默认 Chrome `user` 会话。
- 优先复用 `127.0.0.1:9222` 上已有 Chrome 标签页。
- 网页微信入口：`https://localhost:3001/`。
- 目标群：`智店通入库群`。

## 禁止事项

- 禁止打开新的浏览器。
- 禁止打开空白浏览器。
- 禁止新建 Chrome Profile。
- 禁止清理登录缓存。
- 禁止退出账号后重登。
- 禁止使用 Browser/in-app browser/browser-use/Playwright/Puppeteer/Chromium launch 打开采集页面。
- 禁止脚本扫 DOM、批量跳页、连点、盲点。

## 必做扫描

1. 先确认当前 Chrome/网页微信已登录且目标群可进入。
2. 从当前会话列表或右侧信息栏入口进入 `智店通入库群`，禁止用搜索栏搜索群名。
3. 从上次成功/阻塞/未扫描时间点开始，先做历史方向扫描。
4. 历史方向必须一直向上翻到上一次已确认的最后一张代扫教育补箱码图片，或明确找到上次已确认扫描边界。
5. 未到达上次箱码边界前，不允许判断无新增或已扫完。
6. 查看群图片优先进入群后打开看到的第一张相关照片，然后用图片查看器左箭头或键盘左键逐张向历史图片方向过图，不靠反复大滚动猜图，不随意左右来回试。
7. 每张图片、缩略图、上传状态卡、箱码截图都必须逐张打开或放大确认。
8. 必须一直向左核对到目标图片、对应箱码图、上传完成卡、核销成功卡和最后一次已经采集过的代扫教育补照片边界。
9. 记录：时间、发送人、图片序号、是否箱码、是否教育补代扫、SKU/PN/SN/型号。
10. 箱码图、外箱标签图、上传完成卡、核销成功卡必须串成同一条记录。
11. 只看到核销成功卡、上传完成卡或代扫文字但未继续用图片左右键找到对应箱码，只能写 `executed_not_closed`。
12. 历史方向扫描后，必须回到最新方向复扫一次；可疑区域再二次回扫。

## 输出

正式记录路径：

```text
apps/inventory-sync/artifacts/manual/education-agent-scan/education-agent-scan-YYYY-MM-DD-HHmm.json
```

阻塞回执路径：

```text
apps/inventory-sync/artifacts/manual/openclaw/receipts/education-agent-scan-YYYY-MM-DD-HHmm.json
```

## 收口标准

- 有新增代扫：必须形成结构化图片清单和代扫记录。
- 确实无新增：必须到达上次箱码边界后，才允许生成同日 `confirmedNoNewRecords`。
- 未到箱码边界、图片清单不完整、无法进入群、登录失效、二维码、白屏、验证码、安全验证：只能 `executed_not_closed` 或 `blocked_page_risk`。
- 采集后必须同步写入 SQL 主链，刷新前端，并检查前端教育补/出入库流水同步可见；未完成 SQL 与前端复核，不能写 `real_completed`。
