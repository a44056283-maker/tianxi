# Claude 任务：报价采集定时任务草案

前置条件：必须先完成 `01-报价采集练习-第1轮.md`，并生成回执。

## 目标

根据练习回执，把“报价采集”整理成可重复执行的定时任务流程草案。

本文件不是让你马上执行正式定时任务，而是让你根据练习结果形成 SOP。

## 微信操作硬规则

正式定时任务草案也必须保留这条边界：

- 微信、公众号、聊天窗口只允许用户人工手动点击。
- Claude 只能下达人工操作指令、读取人工保存的文件或截图、判断证据、写回执。
- Claude 不得运行脚本去点击微信。
- Claude 不得用 AppleScript、坐标点击、UI 自动化、Computer Use、浏览器自动化控制微信。
- Claude 不得自动点击 `Allow Once`。
- Claude 不得后台抓取微信数据。

如果正式流程仍会触发 `Allow Once`，则该流程只能标记为 `blocked_permission_prompt`，不能标记为已自动化。
