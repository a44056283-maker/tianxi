# 第一步：可见界面状态识别

用途：先验证 Claude 是否能按手动采集要求识别当前桌面界面。此任务只做状态识别，不做数据写入。

## 必读规则

执行前读取：

```text
docs/claude-scheduled-tasks/README.md
docs/claude-scheduled-tasks/00-AllowOnce权限处理规则.md
docs/claude-scheduled-tasks/03-循环执行协议.md
docs/claude-scheduled-tasks/04-可见界面报价采集到数据对接.md
```

## 只做什么

只完成第一点：

1. 识别当前桌面是否能看到微信。
2. 判断微信是否已登录。
3. 判断当前是否在分销群、公众号、文件列表、聊天列表或其它页面。
4. 判断是否能看到今天报价文件、报价原文或报价截图。
5. 判断是否存在 `Allow Once`、登录、验证码、风控、白屏、权限弹窗。
6. 写状态回执。

## 不做什么

- 不下载文件。
- 不点击报价文件。
- 不打开公众号文章。
- 不解析 Excel。
- 不写入 `public/data`。
- 不写入 SQLite。
- 不重建快照。
- 不验证前端。
- 不使用脚本、AppleScript、shell、Python、Node、坐标点击或后台进程操作微信。
- 不自动点击 `Allow Once`、登录确认、验证码、安全验证或权限弹窗。

## 输出回执

写入：

```text
docs/claude-scheduled-tasks/receipts/YYYY-MM-DD-第一步-可见界面状态识别.md
```

回执必须包含：

```text
结论：
- ready_for_manual_collection / blocked_page_or_login / blocked_permission_prompt / blocked_missing_input

当前界面：
- 是否看到微信
- 是否已登录
- 当前页面类型
- 是否看到今天报价入口
- 是否有权限/登录/验证码阻塞

下一步：
- 如果 ready_for_manual_collection，说明下一步可以执行哪个手动采集动作。
- 如果 blocked，说明用户需要先做什么。
```
