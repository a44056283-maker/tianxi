用途：让 Claude 按真实可见界面完成报价采集，并把采集结果接入联想智慧零售系统。

## 总目标

不是只读旧文件，也不是只写回执。

本任务要求完成完整闭环：

```text
可见界面采集 -> 保存原始证据 -> 解析报价 -> 写入项目数据 -> 重建快照 -> 验证前端 -> 写回执
```

## 手动采集对接原则

本任务是“手动可见界面采集 + 软件数据对接”，不是脚本后台采集。

必须遵守：

- 只能在可见桌面界面里人工式操作，先看清当前页面，再进行下一步。
- 不得写脚本抓微信数据。
- 不得用 AppleScript、shell 脚本、Python、Node、坐标点击脚本或后台进程操作微信。
- 不得后台读取微信数据库、缓存、Cookie、登录态或容器私有数据。
- 不得自动点击 `Allow Once`、登录确认、验证码、安全验证或权限弹窗。
- 采集到的 Excel、截图、文本必须先保存到项目人工输入目录，再解析和写入软件数据。
- 数据对接只能发生在原始证据保存之后，不得无证据直接写正式快照。

## 采集对象

优先级：

1. 微信分销群当天报价 Excel。
2. 灰渠公众号当天报价原文或截图。

日期口径：以今天为准，不得把历史报价写成今日新采。

## 执行流程

### 1. 识别当前桌面状态

先判断：

- 微信是否已打开。
- 是否在目标群或公众号页面。
- 是否已经能看到当天报价文件、原文或截图。
- 是否出现登录、验证码、权限确认、Allow Once、风控、白屏。

不得跳过状态识别直接采集。

### 2. 可见界面采集

如果微信已登录且页面可操作：

- 通过可见界面打开目标分销群或公众号。
- 通过可见界面找到今天的报价文件、报价原文或报价截图。
- 通过可见界面保存到项目人工输入目录：

```text
docs/claude-scheduled-tasks/manual-inputs/
```

建议文件名：

```text
distributor-quotes-YYYY-MM-DD.xlsx
gray-wholesale-YYYY-MM-DD.txt
gray-wholesale-YYYY-MM-DD.png
```

### 3. 权限与风险处理

如果遇到以下情况，停止当前风险路径，写阻塞，不要硬闯：

- 微信未登录。
- 需要验证码、滑块、安全验证。
- 出现 `Allow Once` 或其它权限确认，且无法在当前权限下继续。
- 页面白屏或加载失败。
- 涉及支付、删除、发货、退款、提交订单、账号权限变更。
- 涉及密码、Token、Cookie、验证码、密钥或登录态。

阻塞状态写：

```text
blocked_page_or_login
blocked_permission_prompt
blocked_missing_input
```

### 4. 解析与写入

采集到原始文件后：

1. 校验文件日期和内容日期是否为今天。
2. 解析报价。
3. 只对库存大于 0 的商品参与匹配。
4. 写入对应快照或补充文件。
5. 不确定匹配不得强写，进入人工复核队列。

### 5. 快照重建

价格或链接写入后，按固定顺序重建：

```bash
cd /Users/luxiangnan/Desktop/联想智慧零售项目/lenovo-smart-retail-cockpit/apps/inventory-sync
npm run build:product-url-locks
npm run build:collection-plan
npm run audit:retail-prices
npm run build:retail-zone
```

不得并行执行。

### 6. 前端验证

重建后检查：

```text
apps/web-cockpit/public/data/latest-distributor-quotes.json
apps/web-cockpit/public/data/latest-gray-wholesale-quotes.json
apps/web-cockpit/public/data/latest-retail-zone-snapshot.json
```

如果本地前端已运行，还要验证：

```text
http://127.0.0.1:5174/
```

确认报价来源、今日状态、carry_forward 标记、匹配数量是否正确展示。

## 禁止事项

- 不得使用脚本、AppleScript、坐标点击、后台进程或数据库读取来替代手动可见界面采集。
- 不得只读旧 JSON 就说完成采集。
- 不得把旧文件重跑说成今日实时采集。
- 不得把文件修改时间当作报价日期。
- 不得伪造价格。
- 不得跳过前端验证就说完成对接。
- 不得在未保存原始证据的情况下直接写正式快照。
