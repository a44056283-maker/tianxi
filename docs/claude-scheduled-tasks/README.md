# Claude 定时任务执行包

用途：这里存放可以直接上传给 Claude 执行的 Markdown 任务文件。

当前原则：

1. 先练习，再形成正式定时任务。
2. Claude 练习阶段只负责采集证据、识别状态、写回执。
3. 练习阶段禁止直接修改业务快照、前端数据和 SQLite。
4. 微信、公众号、聊天窗口相关动作只允许人工手动点击；Claude 不得触发脚本、AppleScript、坐标点击、自动化工具去点击微信。
5. Codex 复核 Claude 的回执后，再决定是否进入正式同步。

## 推荐执行顺序

1. 先上传 `00-AllowOnce权限处理规则.md` 给 Claude。
2. 再上传 `03-循环执行协议.md` 给 Claude。
3. 再上传 `01-报价采集练习-第1轮.md` 给 Claude。
4. 等 Claude 输出回执。
5. 把 Claude 回执放入 `receipts/`，或直接发回给 Codex 复核。
6. 复核通过后，再上传 `02-报价采集定时任务草案.md`。
7. 要验证“Claude 自己可见界面采集并接入软件”，上传 `04-可见界面报价采集到数据对接.md`。
8. 如果要先完成最小一步，上传 `05-第一步-可见界面状态识别.md`，只让 Claude 识别当前微信/报价界面状态。
9. 如果要执行完整闭环并在成功后上线后续定时采集，上传 `06-完整报价采集并上线定时任务.md`。

## 循环执行协议

Claude 收到任务文件、路径、项目或目标后，默认直接开始执行，不再问“是否开始”“是否继续”“你希望我做什么”。固定循环为：

1. 读取规则和上下文。
2. 检查任务队列、输入文件和证据。
3. 执行当前任务。
4. 验证结果。
5. 写回执或总结。
6. 继续下一项，直到任务完成、队列清空或出现真实阻塞。

只有自己无法解决、缺少必要输入、登录、验证码、支付、删除、权限变更、真实账号风险时，才可以询问用户。

## 状态口径

只允许使用这些状态：

- `real_completed`：真实完成，有当天证据。
- `executed_not_closed`：执行了，但证据或写回链路未收口。
- `blocked_missing_input`：缺当天原始文件、截图或原文。
- `blocked_page_or_login`：页面、登录、微信、风控或验证码阻塞。
- `blocked_permission_prompt`：出现 `Allow Once` 或其它权限确认，不能无人值守自动化。
- `carry_forward_candidate`：只适合沿用旧有效报价，不能算当天新采。

## 微信操作红线

Claude 可以做：

- 读取本地文件。
- 判断已有证据是否为当天。
- 写练习回执。
- 告诉用户需要手动打开哪里、点击哪里。

Claude 不可以做：

- 不可以运行脚本去点击微信或公众号。
- 不可以用 AppleScript 控制微信点击。
- 不可以用坐标点击、自动化点击、UI 自动化工具操作微信。
- 不可以自动点击 `Allow Once`。
- 不可以后台抓微信数据。
- 不可以在未人工确认界面状态时继续推进采集。
