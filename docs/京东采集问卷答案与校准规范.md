# 京东采集问卷答案与校准规范

## 已确认店铺入口

采集按以下顺序逐店铺尝试。如果前一个店铺没有目标商品或目标配置，再进入下一个店铺补充。

1. 联想京东自营旗舰店  
   `https://lenovo1.jd.com/`
2. 来酷智生活官方旗舰店  
   `https://mall.jd.com/index-11713475.html`
3. 联想官方旗舰店  
   `https://mall.jd.com/index-12894711.html`
4. 联想拯救者官方旗舰店  
   `https://mall.jd.com/index-935158.html`

`https://mall.jd.com/index-1000000157.html` 从采集入口地址库删除；同一联想自营体系统一使用 `https://lenovo1.jd.com/`。

以上 4 个店铺都允许作为备用价来源，但必须记录来源店铺和 URL。

## 采集路径

标准路径：

```text
店铺首页 -> 系列菜单 -> 英雄卡 -> 立即购买 -> 商品详情页
```

禁止把京东搜索框作为主流程。搜索只允许作为人工找链接兜底线索，不能直接写入正式价格。

禁止使用京东全站顶部分类、顶部导航、首页全站频道、搜索推荐、为你推荐等入口采集。京东采集必须从店铺首页进入，只使用店铺自己的左侧分类、店铺内分类、系列菜单和店铺内英雄卡。不要点“京东首页”“全部商品分类”“电脑馆”“国家补贴”等全站入口，这类入口更容易跳出店铺链路并触发 403。

当前校准后的临时执行路径：

```text
店铺首页 -> 系列菜单/活动页 -> 英雄卡
```

- 先采英雄卡上的标题、卡片价、活动文案和能直接识别到的 `item.jd.com` URL。
- 本机当前京东账号不再二次打开商品详情页校验，二次校验由人工或另一台电脑账号处理。
- 能匹配库存 SKU 的卡片记录允许先以 `provisional` 状态写入首页库存零售专区，便于页面先展示正常。
- 不能通过右键、点击“立即购买”、新开详情页等方式强取 URL；这些动作已经确认可能触发 `reason=403`。

已废止：上述“活动页/英雄卡”临时路径停止使用。活动页大图卡片、营销图形卡片、英雄大图、活动会场卡片一律禁用，不作为京东采集入口，不写入 URL 锁定库，不写入价格来源。京东主流程只采店铺首页左侧分类进入后的标准商品列表，列表必须能直接呈现商品标题、配置、价格和商品链接。

## SKU 匹配规则

库存字段优先顺序：

1. 商品名称型号
2. 规格配置
3. PN/MTM

PN/MTM 只作为备选校验，因为网店数字和库存 PN/MTM 可能不一致。

商品匹配识别顺序：

1. 系列
2. CPU
3. 显卡
4. 内存
5. 硬盘
6. 屏幕尺寸/刷新率
7. 颜色

达到以上顺序中的 3 项有效匹配，即可作为参考实时价格候选。达到条件后停止继续扫描该 SKU，避免参数过多造成不确定性和流量浪费。

第一轮锁 URL 后仍需要人工二次检验。人工确认前，候选价格不得作为完全自动采信价格。

## 价格口径

主价格：

- 采集国补前展示价。
- 不采 PLUS 价、券后价、服务价、升级价作为主价格。

单独记录活动：

- 平台满减
- PLUS 立减
- 国家补贴
- 教育补贴
- 企业会员补贴

排除项：

- 延保
- 服务包
- 以旧换新
- 加硬盘
- 升级硬件
- 其它附加服务类价格

当页面同时出现类似 `¥10944.01` 和 `¥10999` 时，主价格取非 PLUS/非优惠后的国补前展示价，例如 `¥10999`。

带小数点的价格：

- 只要读取到带小数点的价格，默认认定为国补后价、券后价、分期/补贴折算价或页面噪声。
- 带小数点价格不得写入主零售价、国补前价和首页库存零售专区主价格。
- 京东详情页主价格应读取商品详情页图片展示位之前显示的国补前整数展示价。
- 如果采集器只读取到小数价格，允许保留 URL 和活动文案，但价格状态必须进入人工复核，不得自动确认为主价格。
- 如果当前只能读取到国补后小数价，但需要先让首页显示参考零售价：按 `国补后小数价 + 国家补贴金额` 反推，再归到合理的 `99` 结尾整数价作为 `provisional` 主价格；同时必须记录原始小数价、补贴金额和反推说明，等待人工二次校验。

## 跳过规则

- 已经存在京东 `captured` 记录、有效 `item.jd.com` 链接和主价格的库存 SKU，不再进入人工重复采集队列。
- 只有每日正常定时采集发现链接失效、无货、下架、转预售、价格异常或人工明确要求复采时，才启用备用店铺链接或重新采集。
- 手工采集过程中每个 SKU 达到匹配条件并写入记录后，立即跳过该 SKU 的其它店铺和其它配置，避免重复浏览造成流量浪费和风控。
- 如果标准列表中没有目标 CPU/配置，例如 `i7-14700HX` 未出现，立即标记为待人工补链/暂不追采，不得长时间盯住单个缺失配置反复翻找。
- 对缺失配置只保留库存 SKU、目标配置、已查店铺和未命中原因；继续处理其它有库存 SKU。
- 一个 `item.jd.com` 链接同时承载多个型号、多个系列或多个差异较大的配置按钮时，默认跳过不采集、不锁 URL。
- 多型号混挂链接只有在卡片默认标题、默认展示价格和库存 SKU 配置完全一致时才允许记录；不能通过点配置按钮强行把同一链接分配给多个库存 SKU。

## URL 锁定规则

- 京东正式 URL 必须是 `https://item.jd.com/数字.html` 或带查询参数的同一详情页。
- 每个库存 SKU 允许锁定多个京东 URL：主 URL、备用 URL、不同店铺 URL。
- 后续刷新价格优先直接打开已锁定详情 URL。
- 如果主 URL 无货、下架、转预售或价格异常，允许使用备用店铺 URL 替换或补充。

## 风控与验证规则

遇到以下情况立即停止：

- 验证码
- 滑块
- 登录
- 安全验证
- 访问受限
- `reason=403`
- 页面异常跳转

停止后等待 5-10 分钟，再重新打开浏览器继续。Codex 不操作验证码、滑块或登录动作。

如果某个动作首次造成 `reason=403`，该动作路径立即禁用，不在同一账号上反复尝试。当前已禁用的动作包括：

- 在活动页英雄卡上右键/点击“立即购买”以获取链接。
- 在本机账号上打开卡片详情页做二次校验。

## 导航问题校准

京东店铺页面顶部出现“京东首页”属于京东全站导航文案，不等于页面已经跳回京东首页。判断是否误跳必须看：

- 浏览器最终 URL
- 页面标题
- 是否存在店铺名称、进店/关注店铺、搜本店、系列菜单、商品卡片或 `item.jd.com` 链接

采集器不得点击顶部“京东首页”导航；只允许使用店铺菜单、系列菜单、英雄卡和商品详情链接。

补充校准：如果页面出现京东全站左侧类目或首页推荐流，说明已经离开店铺采集链路，应立即返回店铺首页或关闭该标签，不在全站首页继续找商品。

## CLI 命名

建议后续分为两个阶段：

```bash
npm run lock:jd-urls
npm run collect:jd-retail-prices
```

`lock:jd-urls` 只锁定 URL 和生成待确认清单。  
`collect:jd-retail-prices` 只刷新已确认 URL 的实时价格。

未完成人工确认前，不允许直接批量刷新全部京东价格。
