# PO / 教育补实物仓模块设计稿

> 状态：已选型，待落正式业务页  
> 最终选型：前端 `方案 B` + 后台 `方案 B`

## 1. 目标

把“参加过营销 PO / 教育补、但机器实物仍在店内”的库存，正式纳入联想智慧零售系统的双层结构：

1. 前端查看层：放在 `库存详情 -> 库存台账` 下，与 `库存总览` 并排为二级书签。
2. 后台管理层：放在 `系统管理` 内部，形成独立的 `PO / 教育补实物仓管理` 模块。

该模块不是演示页，也不是单独飘在系统外的临时工具，而是要接现有 `retail_core / inventory_movement / serial_item / published projection / terminal views` 主链。

---

## 2. 设计读法

阅读口径：

- 这是一个门店 ERP / 进销存子模块，不是营销驾驶舱。
- 目标用户是门店老板、店长、运营、收银端查看人，而不是访客。
- 视觉方向优先“可操作、可追溯、可审计”，不是卡片化营销展示。
- 推荐视觉语言：`管家婆进销存型`。
- 明确禁止：大方块卡片式占位、英雄卡式比稿、空旷驾驶舱 KPI 宫格。
- 页面必须更接近京东/联想后台订单管理的紧凑式样：工具条、筛选栏、表格、作业区同层收口。

---

## 3. 模块拆分

### 3.1 前端查看页

位置：

- `库存详情 -> 库存台账 -> PO / 教育补实物仓`

职责：

- 只读展示，不提供管理动作。
- 展示门店库存、实物仓库存、合计库存、SN、状态、销售单链、智惠服务单链。
- 明确说明“门店库存和实物仓库存各计各的，但终端可显示合计数量”。

### 3.2 后台管理页

位置：

- `系统管理 -> PO / 教育补实物仓管理`

职责：

- 接收已有 `PO` 销售出库订单。
- 执行实物仓入库。
- 执行库存退出 / 转回门店。
- 执行智惠服务同步出库。
- 执行撤销误转仓 / 撤销误核销 / 服务单重绑。
- 展示审计流水和异常原因。

## 4. 已选模板

## 4.1 前端查看页：方案 B

理由：

- 数字密度高，适合门店真实库存查看。
- 能直接把“门店库存 / 实物仓库存 / 合计 / 待核销”放进同一层读数。
- 比现在驾驶舱卡片更饱满，不会空。

结构：

1. 顶部四个 KPI
   - 门店库存
   - 实物仓库存
   - 合计库存
   - 待核销
2. 中部状态块
   - `active / consumed / released`
   - 销售单 / 智惠服务单链说明
3. 下方实物仓台账表
   - 商品
   - 门店库存
   - 实物仓库存
   - 合计
   - SN
   - 状态
   - 来源销售单
   - 智惠服务单

### 4.2 后台管理页：方案 B

理由：

- 最贴合门店 ERP 管理习惯。
- 能把动作区、状态区、候选池、审计流水压在一个页面里。
- 后续可自然扩展批量操作。

结构：

1. 顶部 KPI
   - 待转仓历史单
   - 活跃持仓
   - 已核销
   - 已退出
2. 候选池
   - 从已有 PO 销售出库单筛选
   - 可选单个或多个商品 / SN
3. 操作区
   - 入仓
   - 智惠服务同步出库
   - 库存退出
   - 撤销误转仓
   - 撤销误核销
   - 重绑服务单
4. 审计流水区
   - 记录动作类型、SN、订单链、操作者、时间、跳过原因

## 5. 前端显示规则

1. 所有终端都必须读取 `physicalHoldStock`。
2. 前端可显示：
   - 门店库存
   - 实物仓库存
   - 合计库存
3. 合计只是显示口径，不改变底层双库存逻辑。
4. 终端显示时必须能区分：
   - 门店正常库存
   - PO / 教育补实物仓库存
5. 进入实物仓的库存，不再重复显示为二次 `营销PO / 教育补` 可申领对象。

---

## 6. 后台业务规则

### 6.1 入仓

来源：

- 只能从已有 `PO` 销售出库订单中选择。
- 支持单个商品或多个商品转入。
- 支持按 SN 精确转入。

本质：

- 这是一个“实物仓入库动作”。

### 6.2 库存退出

来源：

- 对已在 `physical_stock_hold` 的 `active` SN 执行退出。

结果：

- 回到门店库存口径。
- 不再占用实物仓数量。

### 6.3 智惠服务同步出库

必须同时满足：

1. `SN` 命中
2. `MTM/PN` 一致

若只命中 `SN`、但 `MTM/PN` 不一致：

- 不允许出库
- 必须记录 mismatch 原因

### 6.4 纠错

必须支持：

- 撤销误转仓
- 撤销误核销
- 重绑智惠服务单

这些动作必须写审计流水，不能做静默覆盖。

## 8. 验收标准

### 8.1 前端查看页

必须满足：

1. `库存台账` 下能看到 `PO / 教育补实物仓` 并排子书签。
2. 页面不是空占位，能展示真实字段。
3. 能看出门店库存、实物仓库存、合计库存三种口径。

### 8.2 后台管理页

必须满足：

1. `系统管理` 内直接管理，不再跳到别的页面。
2. 入仓、退出、核销、撤销、重绑操作都有独立区域。
3. 候选池、动作区、审计流水能同页完成。

### 8.3 链路

必须满足：

1. 入仓能落到 `physical_stock_hold`
2. 退出能更新 `inventory_movement / serial_item / physical_stock_hold`
3. 智惠服务同步出库必须按 `SN + MTM/PN` 校验
4. 所有终端同步显示实物仓库存
5. 不重复计算 `营销PO / 教育补`
