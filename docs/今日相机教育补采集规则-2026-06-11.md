# 今日相机教育补采集规则

日期：2026-06-11

## 1. 目标

从今天起，教育补代扫采集不再以微信群图片回扫作为正式采信来源，改为以今日相机团队相册/API 同步照片作为主采集源。

微信群历史图片：
- 不再作为 `zhidiantong-sync-cycle` 的前置门禁。
- 不再要求“两个群逐图扫到历史边界”才能判定当天同步完成。
- 如后续仍需查看，只能作为历史补证，不再作为正式主链。

## 2. 前置条件

要进入正式链路，必须满足以下条件：

1. 已拿到今日相机开放平台 `groupKey` / `groupSecret`。
2. 上传位置是团队相册/团队照片，不是个人私有相册。
3. 门店端按固定模板上传，不允许随意拍、随意命名、随意漏图。

API 的有效期目前未知，因此每天首次运行前要增加一次轻量健康检查：
- 校验凭证是否可调用。
- 校验最近 24 小时是否能读到团队照片。
- 失败时写成 `blocked_missing_input`，原因明确写“今日相机 API 凭证失效或未开通”。

## 3. 采集来源定义

教育补代扫记录统一允许以下来源类型：

- `xhey_api_manual`
- `watermark_camera_manual`

兼容历史数据时，旧来源 `wechat_group_manual` 继续可读，但只作为历史记录，不再作为新规则推荐来源。

## 4. 记录文件

当天采集结果仍落到：

`apps/inventory-sync/artifacts/manual/education-agent-scan/education-agent-scan-YYYY-MM-DD-*.json`

但字段语义调整为：

- `sourceType`：真实采集来源，优先写 `xhey_api_manual`
- `collectionSource`：固定写今日相机相关来源，例如 `xhey_api`
- `sourceGroupName`：当前只保留为费率归类字段，不再表示微信群来源

现阶段 `sourceGroupName` 仍允许：

- `智店通入库群`
- `教育补贴群`

这两个值现在只决定费率口径：

- `智店通入库群` -> 50 元/台
- `教育补贴群` -> 30 元/台

后续如果费率规则彻底去群化，再单独改字段名。

## 5. 单条记录必填字段

每条正式记录至少要有：

1. `sourceType`
2. `sourceGroupName`
3. `collectionSource`
4. `productName`
5. `modelText`
6. `serialNumbers`
7. `customerName`
8. `customerPhone`
9. `agentPhone`
10. `voucherCode`
11. `voucherVerifiedAt`
12. `reportStatus`
13. `scanDate`
14. `orderNumber`（能匹配时必须写）

对于今日相机来源，额外要求保留原始证据字段：

1. `photoId`
2. `mediaUrl`
3. `takenAt`
4. `userId`
5. `userName`
6. `watermarkText`

如果现有 JSON 结构暂时没有这些字段，先放进扩展对象或备注字段，但不能丢。

## 6. 图片规则

普通单扫至少需要：

- 1 张产品信息图
- 1 张教育优惠券核销码图

两件套至少需要：

- 2 张产品信息图
- 3 张核销码图

三件套至少需要：

- 3 张产品信息图
- 4 张核销码图

箱码图不再作为主采信图，只能做辅助证据。

## 7. 订单归并规则

以下任一条件成立，即可归并为同一订单代扫：

- 客户电话一致
- SN 一致
- 姓名一致

如果 `customerPhone` 与 `agentPhone` 不一致，必须继续生成电话不一致告警，不得静默通过。

## 8. 套装规则

以下规则保持不变：

1. 三件套、两件套、拯救者双屏畅玩两件套，必须命中：
   - 营销库 MTM 物料号
   - 同一真实销售出库单号
2. 套装代扫费只能按整单计算一次。
3. 命中套装后，不能再回落成单品代扫重复收费。

## 9. 当天无新增

如果当天确认没有新增代扫记录，可以生成 `confirmedNoNewRecords`。

但要求：

1. `sourceType` 必须是今日相机来源。
2. `collectionSource` 必须明确写今日相机来源。
3. 不能再用“微信群已扫到边界”作为无新增依据。
4. 依据应改为“今日相机当天时间窗口内无符合规则的新照片”。

## 10. 与调度任务的关系

从本规则生效起：

1. `zhidiantong-sync-cycle` 不再因为“微信群当天没扫”单独失败。
2. 教育补仍然要检查：
   - `education-agent-scan-sync-gap`
   - SQL 是否已写入
   - 前端是否已可见
3. 真正阻断主链的条件，只保留“教育补真实缺口未清空”，不再保留“微信群未回扫完成”。

## 11. 下一步实现建议

1. 先补一个今日相机凭证健康检查。
2. 再补一个“今日相机照片 -> OCR/字段提取 -> education-agent-scan JSON”的正式导入器。
3. 最后把 `sourceGroupName` 从“群名”重命名为“费率口径/结算口径”。
