## 总原则

1. 群报价每天到点都必须扫描原始文件；如果发现更新就同步，没有更新就不重复同步。
2. 灰渠公众号报价继续按上午检查、午后补查执行。
3. 现有群报价不是当天时，按每日流程查找当天原始文件并解析群报价。
4. 有当天灰渠公众号原文时，才更新灰渠批发价。
5. 没有新输入就明确写 `carry forward` 或 `blocked_missing_input`。
6. 不得把旧报价写成当天新采。
7. 任务必须收口，不能停在“已运行但没结论”。每轮只能收口为：
   - `real_completed`
   - `blocked_missing_input`
   - `blocked_page_risk`
   - `scheduler_not_triggered`
8. 如果当天真实输入已经存在且登录状态正常，任务必须完成同步、重建和前端检查。
9. 如果任务失败，必须同轮重试一次；仍失败时必须写清失败文件、失败步骤、错误信息和需要人工处理的动作。

## 一、分销商群报价更新教程

### 任务名

- `daily-price-channel-check`

### 真正完成判定

当前口径：群报价仍是每日实时进货价主链；每天到点都要扫描，发现更新就同步，没有更新就不重复覆盖。

同时满足以下条件才算“今天群报价已真实可用”：

1. 已扫描分销群报价原始文件位置
2. 现有 `latest-distributor-quotes.json` 已经同步到最新当天原始文件，或任务成功找到并解析新的当天原始文件
3. 任务执行结果为 `real_completed`
4. 前端派生快照已刷新

### 原始输入位置

如果现有快照不是当天，任务应按每日流程查找当天群报价文件，只允许以下当前入口：

- `apps/inventory-sync/artifacts/manual/wechat-quote-collection/current/`
- `~/Downloads/codex-installs/wechat-selkies/config/xwechat_files/`
- Chrome `https://localhost:3001/` 网页微信下载得到的当天文件

如果这些位置没有当天文件，再去 Chrome `https://localhost:3001/` 网页微信可见界面确认；禁止打开微信桌面版。

### 执行步骤

1. 进入项目根目录：

```bash
cd /Users/luxiangnan/Desktop/联想智慧零售项目/lenovo-smart-retail-cockpit
```

2. 执行任务。该任务每天都会先扫描原始文件；发现新当天文件才解析同步：

```bash
bash scripts/run_scheduled_task.sh daily-price-channel-check
```

3. 读取结果：
   - `apps/inventory-sync/artifacts/latest-scheduled-task-reports.json`
   - `apps/web-cockpit/public/data/latest-distributor-quotes.json`

### 本轮要看的关键字段

1. `executionOutcome`
2. `updatedRecordCount`
3. `unmatchedProductCount`
4. `evidencePaths`

### 常见问题

#### 1. 现有群报价快照不是今天，且找不到当天原始文件

- 只能写：
  - `blocked_missing_input`
- 不能写“群报价已更新”

#### 2. 文件有了，但不是今天日期

- 只能写：
  - `executed_not_closed`
  - 或 `blocked_missing_input`

#### 3. 任务入口报 `tsx / listen EPERM`

- 当前脚本已补自动 fallback
- 仍统一走：

```bash
bash scripts/run_scheduled_task.sh daily-price-channel-check
```

## 二、灰渠公众号报价更新教程

### 任务名

- `daily-gray-channel-check`

### 真正完成判定

同时满足以下条件才算“今天公众号报价已真实更新”：

1. 找到当天公众号原文或当天有效截图
2. 原文已保存到 manual 文件
3. 任务执行后 `executionOutcome = real_completed`
4. `quoteDate = 当天`
5. `isCarriedForward = false`

### 当前真实边界

如果当天没有新原文，就只能写：

- `blocked_missing_input`

不能为了“每天都要更新”就把旧公众号报价冒充成今日新价。

### 原始输入优先级

1. 当天公众号原文
2. 当天有效截图
3. 如果都没有：
   - 沿用旧值
   - 明确阻塞

### 执行步骤

1. 进入项目根目录：

```bash
cd /Users/luxiangnan/Desktop/联想智慧零售项目/lenovo-smart-retail-cockpit
```

2. 执行任务：

```bash
bash scripts/run_scheduled_task.sh daily-gray-channel-check
```

3. 读取结果：
   - `apps/inventory-sync/artifacts/latest-scheduled-task-reports.json`
   - `apps/web-cockpit/public/data/latest-gray-wholesale-quotes.json`

### 本轮要看的关键字段

1. `executionOutcome`
2. `blockingReason`
3. `quoteDate`
4. `isCarriedForward`
5. `carryForwardFrom`

### 常见问题

#### 1. 没有当天原文

- 这是正常阻塞
- 统一写：
  - `blocked_missing_input`

#### 2. 只有旧截图

- 不能冒充今天更新
- 只能继续沿用旧值并标注来源日期

#### 3. 微信/公众号掉线

- 统一写：
  - `blocked_page_risk`

## 四、更新顺序固定规则

以后每次手工补报价，顺序固定为：

1. `daily-price-channel-check` 每天先扫描群报价原始文件；有新当天文件就同步，没有更新就不重复同步
2. `daily-gray-channel-check` 检查灰渠公众号当天原文
3. 重建并检查前端快照
4. 检查任务报告
5. 检查库存详情、零售专区、防流失补贴价是否读取实时灰渠链路
6. 再决定是否继续补页面证据
7. 如果前端任一关键位置没有显示新同步结果，本轮不得报完成，必须回到解析、映射或快照重建环节修正

## 五、禁止

1. 不得把没有当天原文的公众号报价写成当日更新
2. 不得把旧 Excel 写成当天群报价
3. 不得只改前端时间戳伪装更新
4. 不得跳过任务报告核查
5. 不得到点不扫、不留报告、不汇报阻塞
6. 不得把 `status=completed` 当成业务完成，必须看 `executionOutcome`
