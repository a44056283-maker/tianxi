# 灰渠公众号报价采集规则

更新时间：2026-05-28

## 目标

把微信公众号 `郑州市创业` 的当日报价，通过手动电脑操控方式稳定落到联想智慧零售系统中，形成：

1. 当天原始证据
2. 当天结构化报价快照
3. 报价来源库中的灰渠报价分类视图

## 真实完成判定

灰渠任务只有在以下条件同时成立时，才能写成“已真实完成”：

1. 已在 Chrome `https://localhost:3001/` 网页微信可见页面确认当天报价内容存在
   - 路径必须是：文件传输助手聊天记录区 -> 固定公众号入口 -> 公众号页最下面带日期的报价快捷入口
   - 日期必须为当天或当前最新有效报价日期
2. 已把当天原文保存到：
   - `apps/inventory-sync/artifacts/manual/gray-wholesale-YYYY-MM-DD.txt`
   - 或 `apps/inventory-sync/artifacts/manual/gray-wholesale-YYYY-MM-DD.md`
3. 已运行 `daily-gray-channel-check`
4. `latest-gray-wholesale-quotes.json` 的 `quoteDate` 为当天
5. `isCarriedForward = false`

如果没有当天原文，只能写：

- `因缺原始数据阻塞`

不能把沿用旧值写成“今日已更新”。

## 当前采集原则

灰渠公众号报价不再追求脚本自动采集，不再以 `MiniMax`、`browser-use`、无头浏览器或任何绕过微信风控的方式为目标。

这一页虽然是公众号专项，但仍然属于“全采集统一动作门禁”的一部分，必须完整继承项目总规则：

1. 先扫描目标当前状态
2. 判断当前是否处于可操作状态
3. 只执行一次手工单步动作
4. 动作后再次扫描状态
5. 状态正确才允许下一步

正式固定为：

1. 手动电脑操控打开或复用 Chrome `https://localhost:3001/` 网页微信可见页面
2. 每一步操作前，先扫描当前网页微信状态
3. 公众号入口固定为网页微信文件传输助手聊天记录区；只从文件传输助手聊天记录下面用户固定放置的公众号报价入口进入，不再搜索公众号。
4. 只有确认当前处于：
   - 已登录主页
   - 文件传输助手聊天记录稳定可见
   - 文件传输助手聊天记录下面的固定公众号报价入口稳定可见
   - 公众号页最下面带日期的报价快捷入口稳定可见，且日期为当天或当前最新有效报价日期
   才允许执行下一次点击
5. 打开文件传输助手聊天记录下面的固定公众号入口后，必须点击公众号页最下面带日期的报价快捷入口；只有该快捷入口打开后的报价页、当天截图或 OCR 证据才算有效输入
6. 每一次点击后必须再次扫描状态，不允许连续点击或双击补救
7. 立即使用项目脚本做 Chrome 网页微信截图和 OCR：
   - `scripts/capture_wechat_window.sh`
   - `scripts/capture_and_ocr_wechat_quote.sh`
   - 文章多屏时使用：
     - `scripts/capture_wechat_article_pages.sh`
8. 基于 OCR 文本整理当天原文到 manual 文件
9. 再由本地解析脚本结构化入库

分销群报价同样遵循这一原则。

## 固定提示词

后续只要是公众号报价采集任务、练习任务、定时任务、交接任务，提示词里必须原样包含：

```text
操作前先扫描目标当前状态，确认当前 Chrome https://localhost:3001/ 网页微信、登录态、文件传输助手聊天记录、文件传输助手聊天记录下面的固定公众号入口、公众号页最下面带日期的报价快捷入口、加载态是否正确。
公众号报价只允许从文件传输助手聊天记录下面的固定公众号入口进入，进入公众号后必须点击页面最下面带日期的报价快捷入口；禁止搜索公众号，禁止打开微信桌面版。
只有文件传输助手聊天记录、固定公众号入口和最下面带日期报价快捷入口稳定可操作时，才允许执行一次手工单步动作。
每执行一步后，必须再次扫描状态，再决定下一步。
出现登录页、二维码、白屏、底部菜单栏收缩、错误公众号、没有最下面带日期报价快捷入口、日期不是当天或当前最新有效报价日期、非目标文章或风控提示，立即停止并写阻塞。
禁止打开微信桌面版；禁止连点、双击、盲点、错误状态下硬操作。
```

## 微信风控保护规则

1. 禁止双击公众号文章卡片。
2. 禁止连续点击同一文章区域。
3. 禁止在正文区、底部菜单区、滚动区域盲点。
4. 进入以下任一状态时必须立即停止：
   - 扫码登录
   - 仅传输文件
   - 二维码登录页
   - 公众号白屏
   - 缩到底部菜单栏后正文无法恢复
   - 会话跳到非目标公众号
5. 以上情况统一记为：
   - `因页面/风控阻塞`
6. 不允许继续乱点尝试恢复。

## 每日执行顺序

灰渠上午检查与午后补查，统一按以下顺序执行：

1. 获取电脑操控槽位
2. 打开或复用 Chrome `https://localhost:3001/` 网页微信
3. 进入文件传输助手聊天记录区
4. 在文件传输助手聊天记录下面打开固定放置的 `郑州市创业` 报价入口
5. 找当天报价图文、截图或原文
6. 进入后优先执行窗口级截图和 OCR
7. 读取 OCR 文本并整理当天原文
8. 保存到当天 manual 文件
9. 执行：
   - `bash scripts/run_scheduled_task.sh daily-gray-channel-check`
10. 读取最新任务报告
11. 只有 `quoteDate = 当天` 且 `isCarriedForward = false` 才算真实完成

## 当前已验证的技术事实

1. `computer-use` 对微信 Accessibility 树仍然过浅，不适合作为深控主链路。
2. 当前稳定链路已经调整为：
   - Chrome 网页微信只负责把目标内容显示出来
   - 文件传输助手聊天记录下面的固定公众号入口作为唯一人工导航入口
   - 项目脚本负责 Chrome 网页微信截图
   - OCR 负责文本提取
   - manual 文件负责当天原文落地
3. 当前文章页如果价格显示为 `40*0 / 5**0` 这类掩码值，不能写入正式报价快照。
4. 遇到掩码价时，任务状态必须保持：
   - `因原文价格被掩码阻塞`
   不能伪造真实价格。
5. 以后每次点击前，先执行一次：
   - `python3 scripts/classify_wechat_quote_state.py`
   再决定下一步动作。

## 禁止

1. 不再把微信报价采集设计成自动化脚本主链路
2. 不得以 `browser-use`、`MiniMax`、无头浏览器、绕风控方式代替人工电脑操控
3. 没有当天原文，不得伪造报价
4. 不能把 `carry forward` 说成日更成功
5. 遇到登录、验证码、页面异常，必须标记阻塞
6. 当天证据只能固定到当天，不允许后补文件覆盖成“当天已采”

## 后续报价来源库规则

灰渠报价进入报价来源库后，只展示联想体系相关产品，其它品牌报价不予展示、不进入前端分类视图。

当前灰渠公众号报价库只保留以下范围：

1. 联想笔记本
2. 联想平板
3. 联想台式机，包含游戏台机、办公台式机、一体机
4. THINK 笔记本
5. THINK 台式机
6. 扬天
7. 瑞天
8. MOTO 手机

特别归类：

1. `斗战者锋7000 / 斗战者 锋7000` 属于来酷/本酷体系台式机，归入 `联想台式机`，不得归入 `MOTO` 或笔记本。
2. `MOTO` 只识别 moto / razr / 摩托手机，不使用单独的 `Edge` 作为品牌判断，避免误伤其它品牌或型号。

明确不展示：

1. 华为 / MateBook
2. 惠普、华硕、戴尔、苹果及其它非联想品牌
3. 不属于上述范围的配件、显示器、打印机、智能生活等杂项报价

前端 `报价来源 -> 公众号报价库` 固定按：

2. 产品系列

展示，并且每个分类块都要显示：

分类页签固定为：

## SKU 同步与灰渠售价生成规则

灰渠公众号报价同步到前端前，必须先匹配智店通产品库 SKU。

1. 公众号报价库必须展示全部筛选后的联想体系灰渠批发价。
2. 能唯一对应到产品库 SKU 的记录，额外显示 SKU，并进入 SKU 同步和库存价格链。
3. 匹配不到 SKU 的记录不强行匹配，但仍保留在公众号报价库中展示批发价、按规则生成的零售价和国补后价。
4. 对应到 SKU 后，前端报价库必须展示：
   - 灰渠批发价
   - 对应零售价
   - 国补后价格
   - 产品库 SKU
5. 品牌分类下面必须继续提供系列细分筛选，避免同一品牌下小新、拯救者、YOGA、来酷、THINK 等混在一起。

灰渠带星报价仅作为估算计算口径，不冒充实数批发价：

1. 原文出现 `47*0 / 12*00` 等带星报价时，`*` 按 `5` 参与计算。
2. 前端必须保留“带星价，按 * = 5 估算”的提示。
3. 带星估算价可以用于生成内部参考零售价和国补后价，但不能写成公众号给出的精确实数价。

灰渠零售价生成规则：

1. 除平板外：
   - `灰渠批发价 * 1.13`
   - 结果统一取 `99` 元结尾
   - 再按现行国补政策生成国补后售价
   - 电脑类国补前售价 `>= 10000` 时固定减 `1500` 封顶，低于 `10000` 时按实际 `15%` 计算，不能把 `9999` 四舍五入成减 `1500`
2. 平板：
   - `灰渠批发价 * 1.15`
   - 结果统一取 `99` 元结尾，作为国补前售价
   - 国补后价 = `国补前售价 * 0.85`
3. 以上计算结果必须跟在灰渠批发价后面展示。
4. 未匹配 SKU 的报价也执行同一套售价和国补计算规则，只是不写入 SKU 同步链。
