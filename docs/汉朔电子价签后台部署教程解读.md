## 1. 文档真实定位

这份 PDF 不是门店运营手册，而是老版本汉朔后台部署手册。

核心对象：

1. `Shopweb`
   - 上层门店商品与价签数据管理系统
2. `ESL-Working`
   - 门店内基站与电子价签控制服务

## 2. 系统结构

文档体现的架构是：

```text
Shopweb
  -> 通过接口下发门店价签任务
ESL-Working
  -> 负责门店内基站与电子价签通讯
AP / 基站
  -> 与电子价签无线通讯
ESL / 价签
```

## 3. 依赖环境

文档要求：

- JDK 1.8+
- Tomcat 8.5.3
- MySQL 5.7+

操作系统口径：

- Windows Server 2008 64bit+
- CentOS 6.7 64bit+

## 4. 最重要的配置项

配置文件：

```text
config/config.properties
```

重点项：

- `httpserver.port=9000`
- `api1.uplink.url=http://localhost:8080/shopweb-webapps/storeCode/stationHandler`
- `apg2.adapter.port=1234`
- `apg1.ftpserver.port=21`
- `apg1.ftpserver.username=hanshow`
- `apg1.ftpserver.password=hanshow`

## 5. 启动方式

命令行：

- Windows:
  - `bin\\eslworking.bat`
- Linux:
  - `bin/eslworking.sh`

服务方式：

- Windows:
  - `bin\\service.bat install`
- Linux:
  - `/etc/init.d`
  - `chkconfig`

## 6. 文档里明确的常见故障

1. `21` 端口被占用
2. `1234` 端口被占用
3. `9000` 端口被占用
4. Windows 防火墙没放通，导致基站接不进来

## 7. 对我们项目的实际意义

对联想智慧零售系统来说，这份文档至少确认了 4 件事：

1. 汉朔不是一个“单接口推价签”的轻量软件，而是一整套本地后台体系
2. 我们真正对接的对象不是价签本身，而是：
   - Shopweb / PriSmart
   - ESL-Working
3. 电子价签要落地，除了商品价格数据，还要考虑：
   - 门店
   - 货架
   - 基站
   - 模板
   - 更新任务
   - 回执状态
4. 若没有厂商提供的 `Shopweb / ESL-Working` 安装包或授权，不能宣称“可直接部署”

## 8. 当前可执行结论

当前这台 Mac + Docker Desktop：

- 可以先搭测试联调环境
- 可以预部署：
  - MySQL
  - Tomcat
  - ESL-Working 目录结构
- 但没有厂商包之前，不能写成“汉朔后台已正式安装完成”
