## 双轨结构

### 方案一：API 主链

链路：

`今日相机 OpenAPI -> 图片/水印拉取 -> 归类规则 -> Bridge API -> SQL -> 前端静态快照`

用途：

- 正式生产采集
- 每日自动同步
- 健康检查
- 可追溯原始来源

优先理由：

- 接口稳定性高于网页抓取
- 可直接拿到成员、照片、水印元数据
- 不依赖浏览器焦点、手动翻页、微信群图片查看器

### 方案二：网页端 CLI 备用链

链路：

`今日相机网页端登录态 -> 浏览器脚本/CLI -> 分类文件夹扫描 -> 下载/整理 -> 同结构 JSON -> Bridge API -> SQL -> 前端`

用途：

- API key 失效
- API 限流
- 开放接口字段不足
- 需要人工核查网页端分类目录

边界：

- 不是主链
- 只作为故障切换和补证入口
- 需要真实登录后再固化 DOM 选择器
