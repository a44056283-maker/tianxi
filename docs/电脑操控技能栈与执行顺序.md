# 电脑操控技能栈与执行顺序

更新时间：2026-05-15

## 目标

把联想智慧零售项目里“电脑操控相关能力”拆成：

1. 当前会话已经可直接使用的内建技能
2. 后续可评估接入的增强能力
3. 各类任务固定先走哪一层

本文件只定义项目内执行顺序，不把第三方 GitHub 项目误写成“已经接入 Codex 可直接调用”。

## 当前已加载并可直接使用

以下能力已经在当前 Codex 会话可用，不需要额外安装：

1. `chronicle`
   - 用途：
     - 判定当前屏幕到底在哪个窗口、哪个页面、哪个对话框
     - 解决“你说的那个页面到底是哪一个”这类歧义
   - 固定位置：
     - 所有电脑操控任务的入口判定层

2. `computer-use`
   - 用途：
     - 控制本机 App
     - 点击、输入、切换窗口、读取可访问性树
   - 固定位置：
     - 微信、桌面程序、系统设置、Finder 等 App 操作层

3. `chrome`
   - 用途：
     - 使用现有登录态、Cookie、扩展和稳定会话
   - 固定位置：
     - 京东、联想官网、智店通、其他依赖真实浏览器会话的页面采集

4. `browser`
   - 用途：
     - 本地站点、本机地址、127.0.0.1、局域网页面验证
   - 固定位置：
     - `5174`、本地 UI、静态快照只读验证

## 固定调用顺序

### 1. 微信 / 文件传输助手 / 公众号 / 群报价

固定顺序：

1. `chronicle`
   - 先确认微信是否在前台、是否仍在正确会话
2. `computer-use`
   - 只做可见界面内的人工辅助操控
3. 本地 manual 文件落地
4. `apps/inventory-sync` 本地解析与快照写回

禁止：

- 不把 `chrome` 当微信替代品
- 不把脚本、无头浏览器、绕风控方案当主链路
- 不把旧 manual 文件重跑写成当天新采

### 2. 京东 / 联想官网 / 智店通

固定顺序：

1. `chronicle`
   - 先确认用户当前到底打开了哪个业务标签或页面
2. `chrome`
   - 进入真实稳定浏览器会话采集或核验
3. 需要桌面级补动作时再用 `computer-use`
4. 采集结果进入本地批次文件
5. 再统一重建快照

### 3. 本地前端 / 远程看板 / 5174 验证

固定顺序：

1. `browser`
   - 打开本地或局域网页面
2. 必要时再用 `chronicle`
   - 看用户桌面当前是否确实停在目标页
3. 如果要切系统窗口或本机 App，再补 `computer-use`

## 外部增强能力的评估顺序

这些不是当前会话已加载技能，只是后续可接入候选。

### 第一优先：OCR / 截图增强

目标：

- 降低微信公众号和群报价取文时对精准点击的依赖
- 让“可见页面 -> 截图/OCR -> 原文落地”更稳定

优先候选：

1. `NormCap`
2. `TRex`

适用范围：

- 微信公众号图文报价
- 群文件截图取文
- 临时价单 OCR

### 第二优先：macOS 控制增强

目标：

- 提升桌面窗口、菜单、控件识别与点击稳定性

优先候选：

1. `Peekaboo`
2. `MacOS-MCP`

适用范围：

- 微信桌面端可访问性不稳定时的补强
- Finder / 系统设置 / 菜单栏等桌面动作

### 第三优先：微信专用桥接

目标：

- 在不破坏项目真实性边界的前提下，评估是否值得接更强的微信桥接层

候选：

1. `WeChat-MCP`

说明：

- 只可视为实验性方向
- 不直接作为本项目生产主链路

## 当前不做的事

1. 不在本轮直接改写 `~/.codex/skills`
2. 不把 GitHub 仓库 clone 下来就写成“技能已加载”
3. 不把第三方能力直接并入正式采集链
4. 不为了“更自动”破坏微信、京东、智店通的真实性边界

## 项目内落地规则

1. 当前“已加载”的定义只包括：
   - 当前会话原生可直接调用的技能与插件能力
2. 当前“候选增强”的定义只包括：
   - 已确认用途
   - 已排定优先级
   - 但还没有真正接入 Codex 调用链
3. 如果后续真的要安装全局技能或外部 MCP：
   - 必须单独记录安装路径
   - 必须记录权限要求
   - 必须做一次真实任务验证
   - 验证通过后才能升级为“已加载”

## 当前结论

对联想智慧零售项目，正式执行顺序固定为：

1. 先用内建：
   - `chronicle`
   - `computer-use`
   - `chrome`
   - `browser`
2. 外部增强按顺序评估：
   - OCR / 截图增强
   - macOS 控制增强
   - 微信专用桥接
3. 在外部增强真正接入前，所有真实业务任务仍按现有内建能力和项目规则执行。
