# 进销存销售端与智店通同步开发边界

更新时间：2026-05-12

## 1. 本对话框开发定位

本对话框后续主要负责“进销存销售端 + 智店通同步 + 本地业务中台”开发。

联想智慧零售网页端的定位：

- 门店零售数据展示端
- 库存、价格、SN、价保、陈旧库存的管理展示端
- 数据回测、经营分析、老板看板、店面管理看板
- 电子价签任务、同步状态、异常复核的管理入口

本对话框负责的进销存销售端定位：

- 本地门店业务事实源
- 门店收银销售录入
- 手机端扫码出库
- 采购入库、销售出库、退货、调拨、盘点调整
- SN 台账与库存流水
- 员工、角色、操作留痕
- 智店通同步任务、失败重试、对账与差异告警
- 未来联想厂家系统和电子价签网关适配

## 2. 系统关系

```text
门店员工 / 手机端 / 收银端
  -> 本地进销存销售端
  -> 本地 SQLite/Postgres 业务中台
  -> 智店通同步任务队列
  -> 智店通 / 联想厂家系统

本地业务中台
  -> 联想智慧零售网页端
  -> 驾驶舱 / 库存报价 / 陈旧库存 / 价保 / 电子价签任务展示
```

核心原则：

- 本地进销存系统先记录真实业务事实。
- 智店通作为外部同步目标和对账对象。
- 联想智慧零售网页端读取本地业务中台和派生快照，不直接承担收银主链路。
- 浏览器自动化只用于审计、补录、字段探索和临时兜底，不作为正式生产主链路。

## 3. 当前工程进度

已完成：

- 前端 `apps/web-cockpit` 已有智慧零售驾驶舱、库存报价、SN 周转、价保、价格采集展示。
- 后端 `apps/api-server` 已有 FastAPI 骨架。
- `apps/inventory-sync` 已有智店通库存、销售出库、采购入库、其他出库同步/导入原型。
- 已确认智店通包含库存、订单、POS、员工、财务、销售报表和 SN 报表等模块。
- 已建立本地优先同步策略：本地先落单，再同步智店通。
- 已新增 SQLite 核心层 `apps/api-server/app/retail_core.py`。
- 已建立初版核心表：商品、SKU、SN、库存流水、销售单、采购单、员工、客户、供应商、同步任务、电子价签任务。
- 已支持从调整后库存快照导入 99 个 SKU 和 226 条 SN 台账。
- 已新增销售单 API：`POST /api/sales/orders`。
- 已新增电子价签更新任务 API：`POST /api/price-tags/update-tasks`。
- 已新增分类树 API：`GET /api/retail-core/category-tree`。
- 已同步分类映射：27 个智店通原始子类、11 个智慧零售展示分类、30 个京东/联想导购子类。
- 已同步现有采购入库、销售出库、非零售出库流水 96 条到进销存核心库。

当前不足：

- SQLite 仍是原型层，没有正式迁移机制。
- 销售出库还缺 SN 可售校验、重复出库校验、库存实时扣减。
- 采购入库 API 还未落地。
- 智店通同步任务还没有独立 worker。
- 智店通“创建订单 / 创建出库单 / 回写结果”的稳定接口清单还未最终确认。
- 前端还没有独立的“门店收银出库”和“同步任务处理台”。

## 4. 后续开发主线

### Phase A：本地进销存事实源

目标：

- 本地系统能独立完成销售、采购、库存流水和 SN 台账。

任务：

1. 统一本地流水类型：`purchase_inbound`、`sales_outbound`、`refund_inbound`、`transfer_inbound`、`transfer_outbound`、`stock_adjustment`。
2. 为销售出库增加 SN 可售校验。
3. 销售完成后更新 SN 状态为 `sold`，更新 SKU 可售数量。
4. 增加采购入库 API，写入采购单、采购行、SN、入库流水。
5. 增加退货入库、库存调整的最小 API。

验收：

- 任一 SN 可追溯入库、销售、退货或调整历史。
- 任一销售单可追溯操作员、SKU、SN、售价和库存流水。
- 不依赖智店通成功，本地也能生成完整业务记录。

### Phase B：智店通同步任务队列

目标：

- 本地进销存事实可稳定投递到智店通。

任务：

1. 完善 `sync_task` 字段：幂等键、优先级、重试次数、下一次执行时间、远端回执 ID、死信原因。
2. 新增同步 worker：只处理 `pending` / 到期 `failed` 任务。
3. 先支持销售单和销售出库任务。
4. 再支持采购入库、其他出库、SN 对账。
5. 所有任务只记录状态和回执，不允许静默覆盖本地事实。

验收：

- 同一个订单重复同步不会在智店通生成重复业务。
- 智店通失败不会阻塞本地收银。
- 失败任务可以重试、死信、人工复核。

### Phase C：联想智慧零售网页端兼容

目标：

- 进销存销售端的数据能被现有智慧零售网页展示、分析和回测。

任务：

1. 保留现有 JSON 快照作为前端兼容派生产物。
2. 后端逐步从 SQLite 生成 `latest-inventory-movements.json`、`latest-adjusted-inventory-snapshot.json`。
3. 前端库存报价模块逐步改读 FastAPI。
4. 新增销售端状态卡：今日销售、待同步、同步失败、SN 异常。
5. 新增异常复核入口：智店通差异、重复 SN、库存不一致。

验收：

- 驾驶舱看到的是本地业务中台的结果。
- 智店通同步失败可以在网页端被发现和处理。

### Phase D：电子价签对接

目标：

- 电子价签作为价格执行终端，与本地报价和库存系统打通。

任务：

1. 建立价签设备、价签绑定、模板、更新任务。
2. SKU 价格变更或店长确认后生成 `price_tag_update_task`。
3. 电子价签网关返回成功/失败状态。
4. 网页端展示待更新、已更新、失败价签。

验收：

- 任一 SKU 可查当前绑定价签。
- 任一价格调整可追溯到价签更新任务。
- 价签失败不会影响本地价格事实，只进入待处理队列。

## 5. 近期执行顺序

下一轮开发建议按以下顺序执行：

1. 给 `POST /api/sales/orders` 增加 SN 可售校验和重复出库拦截。
2. 新增 `POST /api/purchases/orders`，支持采购入库写入 SN。
3. 新增 `GET /api/sync/tasks` 和任务状态更新接口。
4. 开始做“门店收银出库”前端页面。

## 6. 不在本对话框优先范围

- 驾驶舱视觉大改版。
- 京东、联想官网、灰渠价格采集细节扩展。
- 广告机内容运营。
- AI 话术生成深度优化。
- 真实账号自动登录、保存 Token/Cookie、绕过验证码。

这些能力可以继续存在于项目中，但本对话框优先推进进销存销售端和智店通同步主链路。
