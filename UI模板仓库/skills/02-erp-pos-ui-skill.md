# 技能 02：ERP / POS 收银后台模式

适用场景：库存详情、SN 台账、入库出库、收银台、商品主档、管理后台、价保汇总、活动管理。

## 学习来源

- 管家婆 iShop：POS 收银、挂单提单、调单退货、会员管理、交班对账、多端应用。
- 管家婆分销 ERP：商品档案、商品类别、计量单位、多编码、客户、供应商、仓库、采购/销售/仓储/财务一体化。

## 执行规则

1. 后台和收银端必须以“台账 + 快捷操作 + 状态反馈”为主，不做营销页。
2. 左侧主模块导航不能删除；右侧顶部放当前模块子流程索引。
3. 时间筛选、关键词检索、分类筛选放在子书签上方，所有子书签共用。
4. 表格是一等公民：出入库、SN、价保、活动历史、报价证据都要台账化。
5. 金额、库存、数量、SN 数量统一右对齐，使用等宽数字。
6. 收银端固定画面优先，1366x768 不应依赖长下拉。
7. 平板横屏采用左分类、中列表、右详情/购物车；手机竖屏采用搜索、分类、卡片、详情弹层。
8. 所有业务数据仍走 SQL/API/受控快照，UI 只做映射，不造数据。

## 禁止

- 前端直接写死假库存、假 SN、假价格。
- 用无限长页面替代分页/页签/详情抽屉。
- 把收银端做成普通网页长列表。
- 把过期活动删除；过期只能从前端默认隐藏，底层历史必须保留。

## 联想智慧零售落地组件

- ERP 左侧模块导航
- 顶部子流程书签
- 统一时间检索条
- 台账表格
- 收银购物车
- 交班对账卡
- 商品主档治理表
