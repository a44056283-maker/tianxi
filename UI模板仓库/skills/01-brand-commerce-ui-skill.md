# 技能 01：品牌电商 UI 模式

适用场景：商品零售、产品英雄卡、商品详情、报价来源、竞品监控、广告机彩页、手机/平板商品查询。

## 学习来源

- Apple 官网/Apple Store：极简层级、服务权益卡、低噪声购买路径。
- 京东自营/联想京东自营：高密度商品卡、价格与活动标签、筛选排序、商品比较。
- 联想官旗商城：商品标题、完整配置、价格区、规格选择、增值服务、详情页签。
- 华为商城：产品族入口、了解更多/购买双操作、服务区、价格提示。

## 执行规则

1. 商品卡必须先展示业务核心：标题、配置、SKU/PN、库存、SN、价格、活动、证据状态。
2. 不在商品区堆长注释；活动、权益、证据全部改成短标签或小卡片。
3. 价格必须分层：门店价、京东、官旗、灰渠、国补后、营销/教育权益。
4. 商品详情用页签：详情、价格证据、活动权益、SN/保修、出入库、历史。
5. 手机和平板不做独立版本，只用同一组件响应式重排。
6. 外部参考价必须标注采集时间和证据状态，不能无证据参与主价。

## 禁止

- 复制第三方页面源码、图片、商品素材。
- 把外部平台标题直接改成系统主标题。
- 用长段解释挤占商品卡首屏。
- 让商品卡在收银机或平板横屏出现横向挤压。

## 联想智慧零售落地组件

- 产品英雄卡
- 商品详情抽屉
- 价格证据卡
- 活动权益条
- 竞品英雄卡
- 移动端商品检索卡
