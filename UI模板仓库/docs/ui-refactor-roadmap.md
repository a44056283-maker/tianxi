# 联想智慧零售 UI 改造路线图

## 目标

基于本模板仓库，将现有零售系统逐步改造成统一、清爽、可自适应的商业软件界面，同时保留真实业务数据链。

## 第一阶段：套壳不动数据

范围：

- 左侧主模块导航
- 顶部子书签 / 子流程导航
- 右侧固定内容区
- 时间筛选统一放到子书签上方

验收：

- `127.0.0.1:5174` 和局域网 IP 显示一致。
- 手机、平板、收银端不出现文字重叠。
- 页面不因单接口失败整页空白。

## 第二阶段：商品零售与库存卡片重构

范围：

- 产品英雄卡
- 库存 / 可售 / SN / 待补 SN
- 京东 / 官旗 / 灰渠 / 门店价
- 营销 PO / 教育补 / 国补权益条

验收：

- 卡片下方可直接显示活动开关和自定义金额。
- 后端活动计算不受前端展示开关影响。
- 没有证据的价格不参与主价计算。

## 第三阶段：ERP 台账化

范围：

- 出入库流水
- 采购入库
- 零售销售出库
- 其他出库
- SN 台账
- 价保历史

验收：

- 所有表格支持统一时间筛选、关键词筛选、分页和导出。
- 所有金额、数量、库存右对齐。
- 数据来源必须从 SQL/API 映射，不直接改前端假数据。

## 第四阶段：收银台独立工作台

范围：

- 商品搜索
- 分类快捷键
- 购物车
- 价格证据
- 活动权益
- 结算、挂单、退货、交班对账

验收：

- 1366x768 收银机固定画面无需纵向滚动。
- 平板横屏保持三栏结构。
- 手机竖屏改为搜索、分类、商品卡、详情弹层。

## 第五阶段：真实系统接入

范围：

- 商品主档
- 门店售价治理
- 活动开关与历史
- 价签 / 收银 / 广告机同步

验收：

- 所有业务字段按 `采集 -> SQL -> API -> 前端` 主链。
- 定时任务不得直接改 UI 文案、主标题和门店零售价。
- 过期活动进入历史库，不清空。

## 后续实现顺序

1. 先改 `apps/web-cockpit/src/App.tsx` 的导航骨架。
2. 再抽出 `RetailShell`、`ModuleTabs`、`KpiStrip`、`ProductHeroCard`、`LedgerTable`。
3. 然后逐个模块迁移：库存详情 -> 产品价保 -> 报价来源 -> 入库出库 -> 收银端。
4. 每迁移一个模块，跑构建并打开真实页面验收。
