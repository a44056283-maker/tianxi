# ERP / POS 模式研究

## 研究边界

本研究用于改造联想智慧零售系统的后台、收银台、库存台账和经营驾驶舱。只学习业务结构和 UI 模式，不复制第三方系统的手册页面或界面素材。

## 1. 管家婆 iShop 可借鉴模式

来源：`https://www.graspishop.com/ishop/`

公开页面明确展示的能力：

- 多端应用：手机端、PDA 端、平板端、电脑端、POS。
- POS 场景：快速收银结算、挂单提单、调单退货、会员管理、交班对账。
- ERP 场景：开单、客户、商品、资金、订货、账单管理。
- 行业能力：批次管理、序列号管理、主副单位、多单位、套装商品。
- 数据实时共享：业务处理、财务记账、数据分析和决策支持。

落到联想智慧零售：

- 收银端不是简化网页，而是固定画面：商品搜索、购物车、价格证据、活动权益、结算快捷键。
- 库存端必须支持 SN 级查询、盘点、异常提示、滞销提醒。
- 管理后台必须支持商品主档、活动开关、门店零售价治理、历史追溯。
- 平板端要适合店员走动查询：大搜索框、分类快筛、商品卡、详情弹层。

## 2. 管家婆分销 ERP A8 TOP 可借鉴模式

来源：公开产品说明书。

公开手册可观察到的信息架构：

- 登录时选择账套和操作员。
- 导航页支持常用功能菜单和可视化报表。
- 基础资料包含商品档案、商品类别、计量单位、多编码、客户、供应商、仓库、职员等。
- 商品档案管理名称、编号、条码、单位、预设价等基础信息。
- 期初建账管理库存期初、金额、应收应付等。
- 分销 ERP 强调采购、销售、仓储、财务一体化。

落到联想智慧零售：

- 主后台必须有“商品主档”而不是只靠前端商品卡。
- 所有采集数据最终要归到 SQL 主链，前端只是映射。
- 出入库、销售、库存、SN、价格、活动、保修要以同一个 SKU 主档贯穿。
- 店员端只展示能操作的入口；管理端展示治理、审计、回放和历史。

## 3. 联想智慧零售目标信息架构

左侧主模块：

1. 今日经营
2. 商品零售
3. 库存台账
4. SN 与保修
5. 产品价保
6. 报价来源
7. 入库出库
8. 收银台
9. 商品主档
10. 系统管理

每个模块顶部子流程：

- 01 概览
- 02 检索
- 03 列表
- 04 详情
- 05 异常
- 06 导出/同步

固定交互：

- 时间筛选放在子书签上方，所有子书签共用。
- 搜索支持 SKU、PN、SN、型号、规格、订单号、客户、金额。
- 表格和卡片共享同一套筛选条件。
- 长内容进入详情抽屉或分页，不把右侧主区域拖成超长页面。

## 4. 收银端布局模板

收银端建议 1366x768 固定画面：

- 左上：商品搜索与分类快捷键。
- 左中：商品结果卡片，支持翻页。
- 右上：当前购物车 / 待结算单。
- 右中：价格证据、活动权益、库存 SN 状态。
- 右下：结算操作、挂单、退货、交班、同步状态。

手机和平板：

- 手机竖屏优先：搜索 -> 分类 -> 商品卡 -> 详情弹层。
- 平板横屏优先：左侧分类 + 中间商品卡 + 右侧详情/购物车。
- 不做多个网页版本，只做同一套组件的响应式布局。
