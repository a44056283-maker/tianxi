# 2026-06-05 UI 深度公开爬取学习档案

## 边界

本轮是公开页面 UI / 信息架构 / ERP-POS 工作流学习，不是生产数据采集。

不做：

- 不复制第三方网站源码。
- 不保存第三方图片、商品素材、价格数据。
- 不绕过登录、验证码或风控。
- 不把公开页面抓到的商品信息写入智慧零售系统业务数据。

输出用途：

- 更新 UI 模板仓库。
- 形成后续改造联想智慧零售系统的设计约束。
- 下一步必须先问用户要改哪个模块，再按 `superpowers/gstack` 写计划执行。

## 爬取源分组

### A. Apple 官网 / Apple Store

公开源：

- `https://www.apple.com.cn/shop/buy-mac`
- `https://www.apple.com.cn/shop/buy-iphone`
- `https://www.apple.com.cn/mac/`
- `https://www.apple.com.cn/store`

观察到的 UI 模式：

1. 极简低噪声页面骨架。
2. 购买页以“选择产品 -> 选择配置 -> 服务权益 -> 购买/配送”推进。
3. 服务权益以短卡片呈现，例如换购、分期、服务支持、课程和专家帮助。
4. 产品族页面先横向展示全系产品，再进入单品购买。
5. 移动端和桌面端保留同一购买逻辑，只改变卡片排列和滚动方向。

可转化为联想智慧零售规则：

- 商品详情首屏只保留决策字段：标题、配置、SKU/PN、库存、SN、门店价、平台价、活动权益。
- 教育补、营销 PO、国补、价保、保修、服务支持全部做短卡片，不做长注释。
- 收银端商品详情采用步骤式右侧抽屉：商品 -> 价格 -> 活动 -> SN -> 结算。

### B. 京东自营 / 联想京东自营

公开源：

- `https://lenovo1.jd.com/`
- `https://mall.jd.com/index-1000000157.html`
- `https://mall.jd.com/index-1000000157.html?from=pc`
- `https://item.jd.com/100089069345.html`

观察到的 UI 模式：

1. 店铺首页以“分类入口 + 商品列表 + 商品比较 + 活动区”组织。
2. 商品列表强调价格、活动、自营/服务标签、评价和可比性。
3. 商品详情页通常把实时价格、促销、配送、规格、库存、服务分成多个异步/模块化区域。
4. 商品详情页的标题、规格组、主要参数、店铺信息和售后服务是独立维度。
5. SKU 规格选择在商品购买判断中非常关键，不能只看主标题。

可转化为联想智慧零售规则：

- 报价来源和竞品监控必须台账化 + 商品卡化：列表用于检索，卡片用于快速销售判断。
- 每个价格信号必须显示来源、更新时间、链接、是否同配、是否可售。
- 同系列不同颜色/内存/硬盘/显卡不能混价；商品卡必须显式展示规格匹配状态。
- 商品比较能力要服务销售：同系列横向比较库存、门店价、平台价、国补后、活动权益。

### C. 联想官旗商城

公开源：

- `https://item.lenovo.com.cn/product/1035434.html`
- `https://item.lenovo.com.cn/product/1048857.html`
- `https://item.lenovo.com.cn/product/1053713.html`
- `https://item.lenovo.com.cn/product/1054293.html`
- `https://item.lenovo.com.cn/product/1029596.html`
- `https://item.lenovo.com.cn/product/1053876.html`

观察到的 UI 模式：

1. 商品详情有面包屑、商品标题、图库、商品编号、商城价、服务说明。
2. ThinkPad / 小新 / 平板 / 一体机在详情结构上保持一致，便于统一模板。
3. 商品编号、商城价、增值税发票、APP 延保、售后服务等被放在购买链路附近。
4. 参数与详情页签对于电脑销售很重要，应该进入可快速查看的详情区域。

可转化为联想智慧零售规则：

- 商品主档必须固定展示 `标准名称 / 完整配置 / SKU / PN-MTM / 商品编号映射`。
- 官旗价格和服务状态进入价格证据，不直接覆盖门店零售价。
- 商品详情必须支持“参数/活动/证据/SN/保修/出入库”页签。
- 一体机、平板、配件的分类要从标题和价格规则共同判断，不只靠旧目录。

### D. 华为商城 / 华为官网

公开源：

- `https://consumer.huawei.com/cn/laptops/`
- `https://consumer.huawei.com/cn/laptops/matebook-14-2024/`
- `https://consumer.huawei.com/cn/laptops/matebook-gt-14/`
- `https://consumer.huawei.com/cn/laptops/matebook-14/select/`
- `https://consumer.huawei.com/cn/laptops/matebook-d-16-2024/`
- `https://consumer.huawei.com/cn/support/content/zh-cn15886706/`

观察到的 UI 模式：

1. 产品族页面先展示系列入口，再进入具体产品。
2. 单品页重视卖点分段、规格参数、服务说明和购买按钮。
3. “了解更多 / 购买”双操作适合同时服务浏览和成交。
4. 服务支持和 Care+ 这类保障信息独立成区，和商品参数分离。
5. 页面反复标注“以实际产品/商城展示为准”，降低价格和参数误导。

可转化为联想智慧零售规则：

- 首页可设置“产品族入口”：游戏本、轻薄本、一体机、台式、平板、配件。
- 详情页同样保留“了解详情 / 去收银 / 看证据”双或三动作。
- 服务、保修、教育补、营销 PO 与商品参数分离，避免混在标题里。
- 外部参考价必须显示证据时间和状态，不让店员误以为是最终成交价。

### E. 管家婆 / ERP / POS

公开源：

- `https://www.gjperp.com/`
- `https://www.graspishop.com/ishop/`
- `https://www.graspishop.com/product/`
- `https://help.wsgjp.com/5e03/d485/163b/ad40/f47f`
- `https://helpnew.wsgjp.com/`
- `https://mkp-res.hc-cdn.com/marketplace/public/app/attachment/20211223/c5630e31-fa61-47e5-a6da-2cd5d58f3e9d/2112230209246208.pdf`

观察到的 UI / 业务模式：

1. ERP 主线是采购、销售、库存、财务、商品资料、客户供应商、仓库和报表。
2. iShop 强调多端：手机、PDA、平板、电脑、POS。
3. POS 收银核心动作：快速收银、挂单、提单、调单退货、会员管理、交班对账。
4. 云商品库和扫码建档适合门店快速新增商品。
5. 帮助中心按业务模块组织：进销存、商品信息、销售出库、采购业务单据、报表、操作视频。
6. ERP 手册强调商品档案、商品类别、基础资料、期初建账、库存预警、采购/销售/仓储一体化。

可转化为联想智慧零售规则：

- 智慧零售系统要有“商品主档”作为所有采集、库存、SN、价格、活动的基准。
- 收银端必须固定支持：商品搜索、扫码、SN 选择、购物车、活动权益、结算、挂单、退货、交班。
- 入库出库必须台账化，默认按时间筛选、单据类型、SKU/SN、供应商、金额检索。
- 库存台账必须支持盘点、陈旧提醒、异常提醒、常卖品告警。
- 后台管理要拆成“商品资料 / 价格治理 / 活动治理 / 同步任务 / 操作审计”。

## 综合模板结论

### 主页面骨架

```text
左侧主导航：今日经营 / 商品零售 / 库存台账 / SN保修 / 产品价保 / 报价来源 / 入库出库 / 收银台 / 商品主档 / 系统管理

顶部子流程：01 概览 / 02 检索 / 03 列表 / 04 详情 / 05 异常 / 06 同步

右侧内容：固定画面 + 分页 + 详情抽屉，不做无限长页面
```

### 商品卡必须字段

```text
标准名称
完整配置
SKU
PN-MTM
分类 / 系列
库存 / 可售 / SN / 待补SN
门店价
京东价
官旗价
灰渠批发价
国补后价
营销PO / 教育补
证据状态 / 更新时间 / 快捷链接
```

### 收银台必须区域

```text
商品搜索
分类快捷键
商品结果卡片
当前购物车
价格证据
活动权益
SN选择
结算 / 挂单 / 退货 / 交班
同步状态
```

### 后台台账必须能力

```text
统一时间筛选
关键词检索
分类筛选
分页
导出
状态标签
金额数量右对齐
证据链入口
异常队列
```
