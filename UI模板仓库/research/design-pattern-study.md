# 电商与品牌站 UI 模式研究

## 研究边界

本研究只提炼公开页面的设计模式。不得复制第三方页面源码、图片、商品数据、品牌素材或完整页面结构。

## 1. Apple 模式

来源：`https://www.apple.com.cn/shop/buy-mac`

可借鉴点：

- 首屏低噪声：大标题 + 少量辅助说明，避免长注释挤占业务区域。
- 购买辅助不是弹窗，而是卡片：换购、分期、翻新、教育等权益拆成可扫读的小卡。
- 横向产品选择适合平板和手机：分类不必全部堆在左侧，核心类别可以做横向可滑动入口。
- 服务和支持并列成模块：把售后、AppleCare、转数据、课程等放在购买后辅助区。

落到联想智慧零售：

- 零售英雄卡顶部不要堆说明文字，保留产品、库存、SN、价格、活动和主操作。
- 价保、教育补、营销 PO、国补等权益做成小型权益条，而不是长段文字。
- 收银端和平板端用横向产品分类条配合左侧主模块，避免单屏过长。

## 2. 京东模式

来源：`https://mall.jd.com/index-1000000157.html?from=pc`

可借鉴点：

- 商品卡必须高密度：标题、价格、活动、服务标签、可比状态同屏出现。
- 筛选和排序强于说明：用户进入商品区后首先要能按分类、价格、库存、活动、品牌筛。
- 自营、配送、售后是信任标签：要用短标签和图标表达，不用长文。
- 商品比较适合多 SKU 决策：电脑销售场景要支持同系列横向比较。

落到联想智慧零售：

- 报价来源、竞品监控、零售报价统一采用“列表筛选 + 英雄卡详情”的组合。
- 价格字段要拆成：门店零售价、京东参考、联想官旗、灰渠批发、国补后、活动备注。
- 商品卡必须能直接看到库存、可售、SN、待补 SN、活动开关、链接证据状态。

## 3. 联想商城模式

来源：`https://item.lenovo.com.cn/product/1043426.html`

可借鉴点：

- 标题下必须有完整配置副标题：CPU、系统、内存、硬盘、显卡、屏幕、颜色。
- 商品详情页将“价格、活动、增值服务、配送、数量、购买操作”分区展示。
- 详情、配置、评价、服务支持用页签组织，避免一页到底。
- 购买按钮和加入购物车是固定高优先级操作。

落到联想智慧零售：

- 产品详情页必须固化“标准标题 + 完整配置 + SKU/PN/MTM + SN/库存状态”。
- 价保专区和商品详情保持同一个页签骨架：详情、价格证据、活动、SN、出入库、保修。
- 不允许把配置缺失的商品直接进入高信任展示区，必须显示证据缺口。

## 4. 华为官网模式

来源：`https://consumer.huawei.com/cn/laptops/`

可借鉴点：

- 产品族首页强调单个旗舰入口，使用“了解更多 / 购买”双操作。
- 顶部搜索和热门产品能快速引导用户。
- 分类快速预览清晰，手机/电脑/平板/耳机/配件保持一致入口。
- 价格提示清楚标注“以商城展示为准”，避免误导。

落到联想智慧零售：

- 首页可保留一个“今日重点产品 / 滞销提醒 / 价保高风险”旗舰区。
- 所有外部参考价格必须标注采集时间和证据状态。
- 店员查询入口要比说明文字更靠前：搜索、分类、价格区间、库存状态。

## 5. 联想智慧零售自有模板方向

视觉基调：

- 白底为主，联想红作主强调，京东式浅蓝灰作辅助。
- 后台/收银区域保持高信息密度，但不要深色大面积压住文字。
- 组件统一 4px 间距基数、8px 圆角、14px 正文、等宽数字。

导航结构：

- 左侧：业务主模块。
- 顶部：当前模块的子流程/子书签。
- 右侧：固定画面内展示当前子书签内容。
- 内容过多时用分页、分栏、弹层详情，不使用无限下拉长页。

核心组件：

- 经营 KPI 条
- 产品英雄卡
- 商品详情抽屉
- 库存 SN 台账
- 价保/活动权益条
- 价格证据表
- 出入库流水表
- 收银快捷操作区
